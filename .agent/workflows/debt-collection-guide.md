# คู่มือการใช้งานระบบติดตามหนี้ (Debt Collection)

ระบบติดตามหนี้ออกแบบมาเพื่อช่วยติดตามออเดอร์ที่ยังไม่ได้รับชำระเงิน (Unpaid) หรือออเดอร์ที่มีปัญหา โดยเน้นการติดตามยอด COD หรือ PayAfter

## 1. การเข้าถึง
- เมนู "ติดตามหนี้" (Debt Collection)

## 2. โครงสร้างหน้าจอ
หน้าจอแบ่งออกเป็น 2 แท็บหลัก:
1.  **กำลังติดตาม (Active)**: แสดงรายการออเดอร์ที่ **ยังติดตามไม่สำเร็จ** (Unpaid) — ไม่รวม BadDebt
2.  **จบเคสแล้ว / จ่ายครบ (Completed)**: แสดงรายการที่ติดตามสำเร็จ (Paid / Partial Paid) หรือจบเคสด้วยเหตุผลอื่น รวมถึง **หนี้สูญ (Bad Debt)**

### Summary Cards
แสดง 2 การ์ดสรุปเหนือตาราง (เปลี่ยนตาม tab ที่เลือก):
- **ยอดค้างชำระทั้งหมด**: ยอดรวม (บาท) ของออเดอร์ใน tab ปัจจุบัน
- **จำนวนคำสั่งซื้อ**: จำนวนออเดอร์ทั้งหมดใน tab ปัจจุบัน

> Summary ดึงจาก `getDebtCollectionSummary()` แยกตาม `status` (active/completed)

## 3. การค้นหาและตัวกรอง (Filters)

### ตัวกรองหลัก (Row 1)
| Filter | รายละเอียด |
|--------|------------|
| **ค้นหา** | ค้นหาด้วยชื่อลูกค้า, เบอร์โทร, หรือ Order ID (ระบบแยกประเภทอัตโนมัติ) |
| **ช่วงวันที่ส่ง** | Date Range Picker สำหรับ `deliveryDate` |
| **ค้างชำระ (วัน)** | Dropdown: ทั้งหมด / เกิน 3, 7, 15, 30 วัน (`minDaysOverdue`) |
| **สถานะการติดตาม** | ทั้งหมด / ยังไม่เคยติดตาม / ติดตามแล้ว |
| **ผู้ติดตาม** | Dropdown รายชื่อผู้ติดตาม (ดึงจาก summary API `trackers`) |

### ตัวกรองเพิ่มเติม (Row 2 - Toggle)
| Toggle | รายละเอียด |
|--------|------------|
| 📦 **ยังไม่ใส่วันรับของ** | กรองเฉพาะออเดอร์ที่ `customerReceivedDate` เป็น null (`missingReceivedDate=1`) |
| 🔴 **ติดตามช้าเกิน 7 วัน** | กรองออเดอร์ที่เลยกำหนดติดตามเกิน 7 วัน (`over7Days=1`) |

- ปุ่ม **"ล้างตัวกรองทั้งหมด"** อยู่ด้านล่างขวา
- เมื่อเปลี่ยน filter ใดๆ → reset กลับหน้า 1 อัตโนมัติ

## 4. คอลัมน์ในตาราง

| คอลัมน์ | รายละเอียด |
|---------|------------|
| **Order ID** | กดเพื่อเปิด `OrderDetailModal`, แสดง Badge "หนี้สูญ" สีแดงถ้าเป็น BadDebt |
| **ลูกค้า** | ชื่อ-นามสกุล จาก `customerInfo` |
| **เบอร์โทร** | คลิกโทรได้ (`tel:` link) |
| **วันจัดส่ง** | `deliveryDate` + แสดง "ผ่านมา X วัน" (`daysPassed`) |
| **วันรับของ** | `customerReceivedDate` หรือ "-" ถ้ายังไม่มี |
| **ยอด** | `totalAmount` + แสดง "ตามได้: X" (`totalDebtCollected`) สีเขียว |
| **ยอดรอตรวจ** | `pendingSlipAmount` (สีส้ม + icon Clock) — แสดงเฉพาะเมื่อ `hasPendingSlips=true` |
| **จำนวนครั้ง** | `trackingCount` — Badge สีเหลือง |
| **ติดตามล่าสุด** | `lastTrackingDate` |
| **ติดตาม** | ปุ่ม "ติดตาม" + แสดงข้อความ "มีการแนบสลิปนอกระบบติดตามหนี้ครบแล้ว" ถ้า `isPendingFullyCovered` |

## 5. สัญลักษณ์และสถานะ
- **หนี้สูญ (Bad Debt)**: Badge สีแดงคำว่า "หนี้สูญ" ใต้ Order ID
- **ยอดรอตรวจ**: ไอคอน 🕐 สีส้ม แสดงยอดสลิปที่รอการตรวจสอบ
- **isPendingFullyCovered**: ข้อความสีส้มเมื่อยอดสลิปรอตรวจครอบคลุมยอดคงเหลือแล้ว

## 6. การดำเนินการ (Actions)

### 6.1 การบันทึกการติดตาม (Track)
กดปุ่ม **"ติดตาม"** เพื่อเปิด `DebtCollectionModal`
- **Order Info**: แสดงรายละเอียดลูกค้าและยอดหนี้คงเหลือ
- **ประวัติการติดตาม (History)**: แสดงรายการติดตามครั้งก่อนๆ (สามารถลบประวัติที่ผิดพลาดได้)
- **แบบฟอร์มบันทึก**:
    - **ผลการติดตาม**:
        - **เก็บไม่ได้**: ติดต่อไม่ได้ หรือลูกค้าไม่ยอมจ่าย (ไม่สามารถอัปโหลดสลิปได้)
        - **เก็บได้บางส่วน**: ลูกค้าจ่ายมาบางส่วน (ต้องระบุยอดและธนาคารของแต่ละสลิป)
        - **เก็บได้ทั้งหมด**: ลูกค้าจ่ายครบ
    - **ยอดเงินที่เก็บได้**: คำนวณอัตโนมัติจากผลรวมสลิป หรือระบุโดยตรง (กรณีรับเงินสด)
    - **บันทึกเพิ่มเติม (Note)**
    - **รูปภาพหลักฐาน (สลิป)**: รองรับหลายรูป, แต่ละรูปต้องระบุยอดเงินและธนาคาร
    - **ต้องการจบเคสนี้ทันที (Close Case)**: ย้ายไปแท็บ Completed
    - **เป็นหนี้สูญ (Bad Debt)**: (ภายใน Close Case) ปรับสถานะออเดอร์เป็น `BadDebt`

> Modal รับ prop `isCompletedView` เพื่อแยก behavior ระหว่าง Active/Completed tab

### 6.2 จบเคส (Quick Close)
- ฟังก์ชัน `handleCloseCase` — จบเคสทันที (ไม่ได้ยอดเพิ่ม)
- บันทึกด้วย `closeDebtCase()` พร้อม `amount_collected: 0, result_status: 3`

### 6.3 ยกเลิกการจบเคส (Cancel Close)
- ฟังก์ชัน `handleCancelCase` — ย้ายกลับจาก Completed → Active
- ทำงาน: ค้นหา history record ที่ `is_complete = 1` → อัปเดตเป็น `is_complete = 0`

### 6.4 ดูรายละเอียดออเดอร์
- กดที่ Order ID → เปิด `OrderDetailModal` แสดงรายละเอียดออเดอร์เต็ม

## 7. ระบบส่งออกข้อมูล (CSV Export)

ปุ่ม **"ดาวน์โหลด CSV"** มุมขวาบน → เปิด Export Modal

### ตัวเลือก Export
| Field | รายละเอียด |
|-------|------------|
| **ประเภทข้อมูล** | 2 ตัวเลือก (toggle) |
| — 📋 ประวัติการติดตาม | แต่ละ record ติดตาม + ผู้ติดตาม + ยอดเก็บ (สำหรับคำนวณค่าคอม) |
| — 📦 ข้อมูลออเดอร์ | สรุประดับออเดอร์ (ยอดรวม, ยอดค้าง) + แยก row ตามสินค้า |
| **สถานะเคส** | ทั้งหมด / กำลังติดตาม / จบเคสแล้ว (required) |
| **วันที่** | start/end — วันที่ติดตาม (history) หรือ วันที่ส่ง (orders) |

### คอลัมน์ในไฟล์ Export

**ประวัติการติดตาม (history)**:
`Order ID, ชื่อลูกค้า, เบอร์โทร, วันที่ส่ง, วันที่รับของ, ยอดออเดอร์, ผู้ติดตาม, วันที่ติดตาม, วันที่ติดตามแรก, ห่างกี่วัน, ยอดเก็บได้, ผลการติดตาม, จบเคส, หมายเหตุ, ยอดเก็บรวม, ยอดคงเหลือ, สถานะออเดอร์, สถานะชำระ, จำนวนเงินในสลิป, ธนาคาร, เวลาโอน`

> **หมายเหตุ**: คอลัมน์สลิป (3 คอลัมน์สุดท้าย) ดึงจากตาราง `order_slips` LEFT JOIN `bank_account`  
> หากมีหลายสลิปต่อ 1 order จะแสดงเป็น pipe-separated (`|`) เช่น `500 | 300`  
> หากไม่มีข้อมูลสลิป จะเว้นว่าง

**ข้อมูลออเดอร์ (orders)**:
`Order ID, ชื่อลูกค้า, เบอร์โทร, วันที่สั่ง, วันที่ส่ง, ยอดรวม, ยอดเก็บแล้ว, ยอดคงเหลือ, วันค้าง, สถานะออเดอร์, สถานะชำระ, สถานะเคส, จำนวนติดตาม, ผู้ติดตามล่าสุด, วันรับเงิน, สินค้าที่ขาย, จำนวนสินค้า`

> **วันรับเงิน**: ดึงจาก `MAX(order_slips.transfer_date)` เฉพาะสลิปที่ผูกผ่าน `debt_collection_images` เท่านั้น (ไม่ใช่สลิปทั้งหมดของ order)

> ไฟล์ CSV ใช้ BOM (\uFEFF) + encoding UTF-8 เพื่อรองรับภาษาไทยใน Excel  
> ชื่อไฟล์: `debt_tracking_history_{start}_{end}.csv` หรือ `debt_collection_{tab}_{start}_{end}.csv`

## 8. ลอจิกการทำงาน (System Logic)

### Pagination
- ข้อมูลเป็น **server-side pagination** ดึงจาก API ทีละหน้า
- ตัวเลือก: 10, 20, 50, 100 รายการต่อหน้า
- Pagination controls แสดงทั้งบนและล่างตาราง

### การค้นหา (Smart Search)
```
searchTerm → ระบบแยกประเภทอัตโนมัติ:
  - ขึ้นต้นด้วย "OD" หรือเป็นตัวเลข ≤5 ตัว → ค้นหาด้วย orderId
  - ตัวเลข >5 ตัว → ค้นหาด้วย customerPhone
  - อื่นๆ → ค้นหาด้วย customerName
```

### Data Fetch
- เมื่อเปลี่ยน tab/filter/page → `fetchOrders()` + `fetchSummary()` ถูกเรียกอัตโนมัติ
- Summary API ส่งคืน `trackers[]` สำหรับ dropdown ผู้ติดตาม

### Tab Logic
- **Active Tab**: `status=active` → ออเดอร์ที่ `is_complete = 0` (ไม่รวม BadDebt)
- **Completed Tab**: `status=completed` → ออเดอร์ที่ `is_complete = 1` หรือ `BadDebt`

## 9. ไฟล์ที่เกี่ยวข้อง

### API Endpoints
| API | หน้าที่ |
|-----|---------|
| `getDebtCollectionOrders()` | ดึงรายการออเดอร์ (paginated) พร้อม filters |
| `getDebtCollectionSummary()` | สรุปยอดค้างชำระ + จำนวนออเดอร์ + รายชื่อผู้ติดตาม |
| `closeDebtCase()` | จบเคสการติดตาม |
| `getDebtCollectionHistory()` | ดึงประวัติการติดตามของ order |
| `updateDebtCollection()` | อัปเดตรายการติดตาม (ใช้ตอนยกเลิกจบเคส: `is_complete=0`) |
| `exportDebtCollection()` | Export ประวัติการติดตาม (type=history) |

### Backend Files
- `api/Finance/get_debt_collection_orders.php`: API ดึงรายการ + summary
- `api/Finance/debt_collection.php`: API บันทึก/แก้ไข/ลบ/export ประวัติการติดตาม
