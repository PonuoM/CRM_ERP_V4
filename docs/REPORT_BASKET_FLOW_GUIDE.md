# 📊 รายงาน Flow ลูกค้าประจำรายเดือน — คู่มือใช้งาน

รายงานสรุป**ลูกค้าประจำในมือ telesale** แต่ละคนต่อเดือน พร้อมแสดงการเปลี่ยนแปลง (คงอยู่ / เพิ่มใหม่ / หลุดออก / เพิ่ม-ลดสุทธิ) เทียบเดือนต่อเดือน

---

## 🎯 รายงานนี้ตอบคำถามอะไร

- ตอนนี้ telesale แต่ละคนมี**ลูกค้าประจำในมือกี่คน**
- เดือนต่อเดือน **เพิ่มหรือลด** และเพิ่ม/ลดจากสาเหตุอะไร
- ใคร **rentain ลูกค้าเก่งสุด** / ใคร **ดึงลูกค้าใหม่ได้เยอะสุด** / ใคร **เสียลูกค้าเยอะสุด**

**นิยาม "ลูกค้าประจำ"**: ลูกค้าที่อยู่ใน basket
- ถัง 39 = **ส่วนตัว 1-2 เดือน** (ลูกค้าที่เคยขายได้แล้วในช่วงสั้น)
- ถัง 40 = **ส่วนตัวโอกาสสุดท้าย** (ลูกค้าที่ยังตามได้)

### บล็อคที่ 1: มี.ค. 69

| คอลัมน์ | ความหมาย |
|---|---|
| อันดับ | ลำดับ (เรียงตาม Baseline ก.พ. มากสุดก่อน) |
| รหัส Telesale | users.id |
| ชื่อ Telesale | ชื่อ-สกุล |
| สถานะ | active / inactive (users.status) |
| **ก.พ. 69 (คน, ตั้งต้น)** | จำนวนลูกค้าในมือ ณ สิ้น ก.พ. — ใช้เป็น baseline |

| **มี.ค. เพิ่มใหม่** | คนที่ไม่อยู่ ก.พ. แต่อยู่ มี.ค. (New) |
| **มี.ค. หลุดออก** | คนที่อยู่ ก.พ. แต่ไม่อยู่ มี.ค. (Lost) |
| **มี.ค. เพิ่ม/ลด สุทธิ** | = เพิ่มใหม่ − หลุดออก (ติดลบได้ = ลูกค้าในมือลดลง) |
| **มี.ค. 69 (คน)** | จำนวนลูกค้าในมือ ณ สิ้น มี.ค. (= คงอยู่ + เพิ่มใหม่) |

### บล็อคที่ 2: เม.ย. 69 และ บล็อคที่ 3: พ.ค. 69
โครงเดียวกัน แต่ baseline เป็นเดือนก่อน (ไม่ต้องแสดงซ้ำ ใช้ "(คน)" ของเดือนก่อนแทน)

**TOTAL row** อยู่ท้ายแต่ละบล็อค — รวมยอดทั้งทีม

---

## 🔍 วิธีอ่าน "เพิ่ม/ลด สุทธิ"

```
ค่า > 0  → ลูกค้าในมือเพิ่มขึ้น     (ดี)
ค่า = 0  → คงที่
ค่า < 0  → ลูกค้าในมือลดลง          (ต้องดูสาเหตุ)
```

**สูตรเช็ค** (มีไว้ verify ข้อมูล):
- เดือนนี้ (คน) = คงอยู่ + เพิ่มใหม่
- เดือนก่อน (คน) = คงอยู่ + หลุดออก
- เพิ่ม/ลด สุทธิ = เพิ่มใหม่ − หลุดออก = เดือนนี้ − เดือนก่อน

## ⚙️ สาเหตุที่ทำให้ "หลุดออก"

ลูกค้าหลุดจากถัง 39/40 ของ telesale ได้ 3 ทาง:

1. **ย้ายถัง (lifecycle)** — โดยเฉพาะ `monthly_cron` ปลายเดือน
   - ลูกค้าในถัง 40 (โอกาสสุดท้าย) ถ้าไม่ปิดดีลก็โดน cron ย้ายไปถัง 54 "พักรอแจก" และถอด assignment
   - ⚠️ **ไม่ใช่ความผิด telesale** — เป็น lifecycle ปกติของระบบ
2. **ถูก transfer** — admin ย้ายลูกค้าให้ telesale คนอื่น (transition_type = `transfer`)
3. **ถูก distribute** — กลับไปสู่ pool กลาง เพื่อแจกใหม่

ดูเฉพาะ "หลุดออก" รวมๆ จะตีความผิดได้ — ควรเช็คใน detail CSV ที่บอก ถังก่อน/ถังหลัง

## 📝 ตัวอย่างการอ่าน — ปุ๋ยดี

```
รวมทีมปุ๋ยดี:
   ก.พ.    มี.ค.   เม.ย.   พ.ค.
   1080 →  1041 →  993  →  942    (ลดลงต่อเนื่อง −138 คน ใน 3 เดือน)
```

### Insight ที่อ่านได้
- **ดาริกา**: มี.ค. ดึงลูกค้าใหม่ได้เก่งมาก (เพิ่มใหม่ 111 คน) — ผลลัพธ์ดี
- **พจนีย์**: เพิ่มใหม่ระดับสูงต่อเนื่อง (121, 38, 31) แต่หลุดออกก็เยอะ (36, 58, 42) — เหมือนคนรับลูกค้าใหม่เยอะแต่ retain ไม่ทัน
- **นางภัควลัญชญ์**: มี.ค. มีปรากฏการณ์ผิดปกติ (หลุด 259 คน) — เช็คประวัติ transition พบว่า **transfer ลูกค้าให้คนอื่น 240 คน** ใน มี.ค. (เหตุการณ์ admin จัดสรรใหม่ ไม่ใช่ performance)
- **ทีมรวม เม.ย.**: หลุดเยอะ (-48) จาก **monthly_cron 191 คน** (lifecycle อัตโนมัติ ทั้งทีมเสียพร้อมกัน)

### Action จากรายงาน
- ดู telesale ที่ "เพิ่ม/ลด สุทธิ" ติดลบติดกันหลายเดือน → คุยเรื่อง pipeline
- ดู telesale "เพิ่มใหม่" ต่ำ → ดูว่ามีลูกค้าใหม่แจกพอไหม
- ดู "คงอยู่" / "เดือนก่อน (คน)" = retention rate → คนที่ rate ต่ำต้องช่วยสอน

## 🔬 เจาะลึกเพิ่มเติม

อยากเช็คว่าลูกค้าที่ "หลุดออก" ในเดือน X ของ telesale T ไปไหน — ใช้ query:

```sql
SELECT transition_type, from_basket_key, to_basket_key,
       assigned_to_old, assigned_to_new, COUNT(*) cnt
FROM basket_transition_log
WHERE assigned_to_old = <T>
  AND from_basket_key IN ('39','40')
  AND created_at >= '<YYYY-MM-01>' AND created_at < '<next month>'
GROUP BY transition_type, from_basket_key, to_basket_key, assigned_to_old, assigned_to_new
ORDER BY cnt DESC;
```

ผลลัพธ์จะบอกว่า:
- transition_type = `transfer` → ถูก admin ย้าย
- transition_type = `monthly_cron` → ระบบหมดอายุอัตโนมัติ
- to_basket_key = '54' → ไป "พักรอแจก"
- to_basket_key = '46' → ไป "หาคนดูแลใหม่"

## 🆘 ปัญหาที่เจอบ่อย

**Q: PHP error "Resource temporarily unavailable" / "fopen failed"**
A: ปิดไฟล์ CSV ใน Excel ก่อน แล้วรันใหม่

**Q: "Unknown column" error**
A: ลำดับ column ใน customers/orders ตาราง schema อาจเปลี่ยน — เช็คกับ `MCP SQL SERVER.md`

**Q: เลขใน summary CSV ไม่ตรงกับ basket count ปัจจุบัน**
A: ตรวจสอบว่าใช้ `tmp_repeat_flow_basket.php` (basket-based) ไม่ใช่ `tmp_repeat_flow_monthly.php` (repeat-buyer based ของเก่า)

**Q: อยากให้รวมถังอื่น (เช่น 38 = ลูกค้าใหม่) ด้วย**
A: ใน SQL ของสคริปต์ เปลี่ยน `IN ('39','40')` เป็นถังที่ต้องการ (ทั้ง 4 จุด: 2 ใน snapshot + ดูใน basket_label map)
