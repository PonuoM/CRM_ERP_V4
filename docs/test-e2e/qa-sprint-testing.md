# 📝 คู่มือการทดสอบ (QA Testing Guide)

เอกสารนี้จัดทำขึ้นสำหรับ QA/Tester เพื่อทดสอบฟีเจอร์ล่าสุดที่เพิ่งได้รับการแก้ไขและพัฒนาเพิ่มเติมในระบบ

---

## 🟢 Feature 1: การบังคับและบันทึกสาเหตุยอดเงินสลิปไม่ตรง (Mismatch Reason)

**วัตถุประสงค์:** เพื่อให้แน่ใจว่าเมื่อยอดเงินในสลิปโอนเงิน (Transfer Slip) ไม่ตรงกับยอดรวมของออเดอร์ (Order Total) ระบบจะบังคับให้กรอกสาเหตุ และสามารถบันทึกข้อมูลสาเหตุนั้น (mismatch_reason) ลงฐานข้อมูลได้อย่างถูกต้อง

### Test Case 1.1: ยอดเงินตรงกัน ไม่ต้องกรอกสาเหตุ
1. **Action:** สร้างออเดอร์ใหม่ หรืออัปโหลดสลิปสำหรับออเดอร์เดิม
2. **Input:** แนบสลิปโอนเงิน และกรอกยอดเงินโอน **ให้ตรงกับ** ยอดรวมของออเดอร์พอดี
3. **Expected Result:** 
   - ระบบ **ไม่แสดง** ช่องให้กรอก "สาเหตุยอดขาด/เกิน"
   - สามารถกดบันทึก/สร้างออเดอร์ผ่านได้ปกติ

### Test Case 1.2: ยอดเงินไม่ตรงกัน ต้องบังคับกรอกสาเหตุ
1. **Action:** สร้างออเดอร์ใหม่ หรืออัปโหลดสลิปสำหรับออเดอร์เดิม
2. **Input:** แนบสลิปโอนเงิน และกรอกยอดเงินโอน **ให้ขาด หรือ เกิน** จากยอดรวมของออเดอร์
3. **Expected Result:** 
   - ระบบ **แสดง** ช่องให้ระบุ "สาเหตุที่ยอดเงินไม่ตรง" (Mismatch Reason)
   - หากพยายามกดบันทึกโดยไม่กรอกสาเหตุ ระบบต้อง **แจ้งเตือน Error/Validation** และไม่ให้ผ่าน

### Test Case 1.3: บันทึกข้อมูล Mismatch Reason ลงฐานข้อมูลสำเร็จ
1. **Action:** ทำต่อจาก Test Case 1.2
2. **Input:** เลือกระบุ "สาเหตุที่ยอดเงินไม่ตรง" (เช่น ลูกค้าโอนขาด, ส่วนลดพิเศษ ฯลฯ) จากนั้นกดบันทึก
3. **Expected Result:** 
   - ระบบบันทึกข้อมูลสำเร็จ
   - เมื่อกลับไปดูรายละเอียดออเดอร์นั้นๆ (หรือเช็คในฐานข้อมูล/หน้ารายการสลิป) ต้องพบข้อมูล **"สาเหตุที่ยอดเงินไม่ตรง"** แสดงอยู่อย่างถูกต้อง ไม่สูญหาย

## 🟢 Feature 2: การเพิ่มลูกค้าใหม่แบบย่อ (Simplified Add Customer)

**วัตถุประสงค์:** เพื่อทดสอบฟังก์ชันหน้าต่าง Pop-up (Modal) การเพิ่มลูกค้าแบบง่าย (ต้องการแค่ ชื่อ, สกุล, เบอร์โทร) ในหน้าจัดการลูกค้า (`ManageCustomersPage`)

### Test Case 2.1: การเปิดหน้าต่าง Modal
1. **Action:** ไปที่เมนู **การจัดการลูกค้า** (Manage Customers)
2. **Input:** คลิกปุ่ม `+ เพิ่มลูกค้าใหม่` 
3. **Expected Result:** 
   - หน้าต่าง Modal "เพิ่มลูกค้าใหม่" เด้งขึ้นมา
   - มีฟิลด์ให้กรอกเฉพาะ: ชื่อ (บังคับ), นามสกุล, และ เบอร์โทรศัพท์ (บังคับ)

### Test Case 2.2: ตรวจสอบ Validation
1. **Action:** ในหน้าต่าง Modal 
2. **Input:** 
   - กดบันทึกโดยปล่อยช่อง "ชื่อ" หรือ "เบอร์โทรศัพท์" ว่างไว้
   - กรอกเบอร์โทรศัพท์ที่มีความยาวไม่ถึง 9 หลัก (เช่น `08123456`)
3. **Expected Result:**
   - ระบบแจ้งเตือนว่า "กรุณากรอกชื่อและเบอร์โทรศัพท์" หรือ "เบอร์โทรศัพท์ต้องมีอย่างน้อย 9 หลัก"
   - ไม่สามารถบันทึกได้

### Test Case 2.3: บันทึกลูกค้าใหม่สำเร็จและรีเฟรชรายการ
1. **Action:** ในหน้าต่าง Modal
2. **Input:** กรอกชื่อ นามสกุล และเบอร์โทรศัพท์ (10 หลัก ตัวเลขเท่านั้น) ให้ครบถ้วน จากนั้นกดบันทึก
3. **Expected Result:**
   - ระบบแสดงสถานะโหลด (กำลังบันทึก...) และบันทึกสำเร็จ หน้าต่าง Modal ปิดลงอัตโนมัติ
   - **(สำคัญ)** ตารางรายชื่อลูกค้าด้านหลัง ต้อง **รีเฟรช (Refresh) ทันที** และแสดงรายชื่อลูกค้าใหม่ที่เพิ่งสร้างขึ้นมาในตาราง โดยไม่ต้องกด F5
