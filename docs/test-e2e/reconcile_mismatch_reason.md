คู่มือนี้อธิบายขั้นตอนการทดสอบ (End-to-End) ระบบการบังคับกรอก "เหตุผลยอดเงินไม่ตรงกัน" (Mismatch Reason) เมื่อพนักงานบัญชีทำการจับคู่ออเดอร์ (Reconcile) กับรายการ Statement ของธนาคารในหน้า Bank Account Audit

## 🧪 Test Scenarios

### Scenario 1: การจับคู่ปกติ (Exact Match)
**จุดประสงค์:** ทดสอบว่าถ้ายอดเงินตรงกัน จะต้องไม่ถูกกวนใจให้กรอกเหตุผล

1. ไปที่เมนู **ตรวจสอบบัญชีธนาคาร (Bank Account Audit)**
2. เลือก Statement ของบัญชีที่ต้องการ
3. ทำการกดปุ่ม "จับคู่" และเลือกรหัส Order ที่ยอดชำระ **ตรงกับ** ยอดเงินเข้าบัญชีพอดี (เช่น ยอดเข้า 100 บาท, ออเดอร์ 100 บาท)
4. เมื่อมีหน้าต่างยืนยัน (Confirm Match) ปรากฏขึ้น
5. **Expected Result:**
   - ระบบต้อง **ไม่แสดง** ช่อง `<textarea>` สำหรับกรอกเหตุผลยอดเงินไม่ตรง
   - สามารถกดยืนยันการจับคู่ผ่านได้ปกติ และสถานะจะกลายเป็นสีเขียว (พอดี)

---

### Scenario 2: การจับคู่แบบยอดไม่ตรงกัน (Mismatch)
**จุดประสงค์:** ทดสอบว่าการเช็คความถูกต้องของส่วนต่างทำงาน และบังคับให้ผู้ใช้ต้องให้เหตุผลที่สมเหตุสมผล

1. ทำการกดปุ่ม "จับคู่" ของ Statement รายการใหม่
2. เลือกรหัส Order ที่ยอดชำระ **ไม่ตรงกับ** ยอดเงิน Statement (เช่น ยอดเข้า 400 บาท แต่ยอดออเดอร์ 500 บาท)
3. เมื่อมีหน้าต่างยืนยัน (Confirm Match) ปรากฏขึ้น
4. **Expected Result 1 (การตรวจสอบ UI):** 
   - ระบบจะแสดงช่อง `<textarea>` แจ้งเตือนข้อความสีแดงว่า **"ยอดเงินไม่ตรงกัน กรุณาระบุเหตุผล *"**
5. **Expected Result 2 (การตรวจสอบ Validation):**
   - ลองปล่อยว่างกล่องข้อความไว้ แล้วกดปุ่ม "ยืนยันจับคู่"
   - ระบบต้องหยุดทำงานและแจ้งเตือนข้อผิดพลาด "กรุณาระบุเหตุผลที่ยอดเงินไม่ตรงกัน" (ไม่ยอมให้บันทึก)
6. **Expected Result 3 (การทำงานสำเร็จ):**
   - พิมพ์ข้อความในช่องเหตุผล เช่น `ลูกค้าขอโอนขาด 100 บาท ไว้จ่ายรอบหน้า` แล้วกดยืนยันจับคู่
   - ระบบจะทำการจับคู่ได้สำเร็จ สถานะจะแสดงเป็นสีเหลือง (ขาด)

---

### Scenario 3: การผูกออเดอร์เพิ่มเติม (Add Additional Order)
**จุดประสงค์:** สำหรับเคสที่ลูกค้าโอนเงิน 1 ยอดมาจ่าย 2 ออเดอร์ เมื่อเพิ่มออเดอร์แล้วยอดรวมไม่ตรง จะต้องเด้งให้ระบุเหตุผลเช่นกัน

1. ไปที่ Statement ที่มีการจับคู่ออเดอร์ไว้อยู่แล้ว 1 ออเดอร์
2. คลิกปุ่ม 3 จุดท้ายรายการ เลือก **"ผูกเพิ่ม (Add Order)"**
3. เลือกรหัส Order อีก 1 รายการ
4. หากยอดรวมของทั้งออเดอร์เก่า + ออเดอร์ใหม่นี้ **มีส่วนต่าง** จาก Statement ยอดเงินตั้งต้น
5. **Expected Result 1:** 
   - เมื่อกดตกลง จะมี `window.prompt()` เด้งขึ้นมาถามว่า `ยอดรวมไม่ตรงกัน (ส่วนต่าง ฿XX.XX) กรุณาระบุเหตุผล:`
6. **Expected Result 2:**
   - ถ้ากด "Cancel" หรือปล่อยข้อความให้ว่างเปล่า ระบบจะแจ้งเตือนและยกเลิกการบันทึก
7. **Expected Result 3:**
   - ถ้ากรอกข้อความเหตุผล แล้วกด OK ระบบจะทำการผูกเพิ่มเข้าไป และรวมยอดใหม่ได้สำเร็จ

---

### Scenario 4: ตรวจสอบการแสดงผลและข้อมูลหลังจับคู่สำเร็จ
**จุดประสงค์:** เพื่อให้แน่ใจว่าระบบบันทึกและแสดงข้อมูลได้ครบถ้วน

1. กลับมาดูที่หน้ารายการ Bank Account Audit 
2. กดดูรายละเอียดการผูก (ไอคอนรูปลูกตา / เครื่องหมาย 3 จุด) หรือนำเมาส์ไปชี้ค้างไว้ที่ป้ายสถานะของ Statement ที่ทำใน Scenario 2 และ 3
3. **Expected Result 1:** 
   - ที่หน้าจอรายละเอียด (หรือ Tooltip) จะต้องมีข้อมูลบรรทัด **"เหตุผลยอดไม่ตรง: ..."** พร้อมแสดงเหตุผลที่เราพิมพ์ไว้เป็นตัวอักษรสีแดงให้เห็นได้ชัดเจน
4. ทดสอบกดปุ่ม **Export CSV** ด้านบน
5. เลือกการดาวน์โหลด
6. **Expected Result 2:**
   - เมื่อเปิดไฟล์ Excel / CSV ขึ้นมา จะต้องมีคอลัมน์ชื่อ **Mismatch Reason** เพิ่มขึ้นมา และบรรทัดที่เราทดสอบต้องมีข้อความเหตุผลปรากฏในคอลัมน์นี้อย่างถูกต้อง
