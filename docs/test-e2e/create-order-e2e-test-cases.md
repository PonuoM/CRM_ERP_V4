# Test Cases สำหรับหน้า CreateOrderPage (สร้างคำสั่งซื้อ / อัปเซล)

**ฟีเจอร์ที่เกี่ยวข้อง**: สร้างคำสั่งซื้อใหม่ (Normal Order), อัปเซลคำสั่งซื้อเดิม (Upsell), อัปโหลดสลิป, ตรวจสอบยอดโอนไม่ตรง (Mismatch Reason)
**ไฟล์**: `pages/CreateOrderPage.tsx`

---

## 🟢 TC1: การสร้างคำสั่งซื้อปกติ (ยอดโอนตรงตามบิล)

**Pre-condition:**
- ผู้ใช้เข้าสู่ระบบด้วยสิทธิ์ Sales
- มีสินค้า, ลูกค้า, โปรโมชั่น และธนาคารในระบบ

**Steps:**
1. ไปที่เมนู "สร้างคำสั่งซื้อ"
2. ค้นหาและเลือกลูกค้าที่มีอยู่ในระบบ หรือกรอกข้อมูลสร้างลูกค้าใหม่
3. กรอกข้อมูลที่อยู่จัดส่ง และเลือกสินค้าลงตระกร้าให้มียอดรวมสุทธิ (เช่น 1,000 บาท)
4. เลือกวิธีชำระเงินเป็น **"โอนเงิน"**
5. อัปโหลดสลิป 1 รูป
6. เลือกระบุธนาคาร, วันที่-เวลาโอน
7. กรอกจำนวนเงินให้ตรงกับยอดสุทธิพอดี (1,000 บาท)
8. สังเกตว่าจะ **ไม่มี** ข้อความแจ้งเตือนสีแดง หรือช่องระบุสาเหตุยอดไม่ตรงบิลปรากฏขึ้น
9. ระบุข้อมูลบังคับให้ครบ (วันที่จัดส่ง, สถานะลูกค้า, ช่องทางขาย)
10. กดปุ่ม "บันทึก"

**Expected Result:**
- สร้างคำสั่งซื้อสำเร็จ 
- สลิปูกบันทึกลงระบบอย่างถูกต้องโดยไม่มี Error

## 🟡 TC2: การสร้างคำสั่งซื้อปกติ (ยอดโอนไม่ตรงตามบิล - ต้องระบุสาเหตุ)

**Pre-condition:**
- ยอดสั่งซื้อสุทธิ 1,000 บาท
- เลือกวิธีชำระเงินเป็น **"โอนเงิน"**

**Steps:**
1. ดำเนินการสร้างคำสั่งซื้อเช่นเดียวกับ TC1 (ข้อ 1-6)
2. กรอกจำนวนเงินในสลิปเป็น **500 บาท** (หรือ 1,500 บาท - ขาดหรือเกิน)
3. สังเกต UI ใต้รายการสลิป
4. กดปุ่ม "บันทึก"
5. กรอกข้อความในช่อง "สาเหตุยอดไม่ตรงบิล" เช่น "ลูกค้ามัดจำ 50% ก่อนจัดส่ง"
6. กดปุ่ม "บันทึก" อีกครั้ง

**Expected Result:**
- **Step 3**: ระบบแสดงกล่องแจ้งเตือนสีแดง (Warning) ว่า "ยอดโอนรวมไม่ตรงกับยอดคำสั่งซื้อ" และแสดงช่องกรอก "สาเหตุยอดไม่ตรงบิล" ปรากฏขึ้นใน Card สลิป
- **Step 4**: ระบบแจ้งเตือน (Alert) "ยอดโอนไม่ตรงกับยอดคำสั่งซื้อ กรุณาระบุสาเหตุที่สลิปอย่างน้อย 1 รูป" และการบันทึกถูกบล็อค
- **Step 6**: สร้างคำสั่งซื้อสำเร็จ ข้อมูลสาเหตุ (`mismatch_reason`) ถูกบันทึกลง Backend

## 🟡 TC3: การสร้างคำสั่งซื้อปกติ (อัปโหลดหลายสลิป)

**Pre-condition:**
- ยอดสั่งซื้อสุทธิ 1,000 บาท

**Steps:**
1. เลือกวิธีชำระเงินเป็น **"โอนเงิน"**
2. อัปโหลดสลิปจำนวน 2 รูป
3. สลิปที่ 1: เลือกระบุธนาคาร, วันเวลา, ยอดเงิน **600 บาท**
4. สลิปที่ 2: เลือกระบุธนาคาร, วันเวลา, ยอดเงิน **400 บาท**
5. สังเกตหน้าจอ
6. กดปุ่ม "บันทึก"

**Expected Result:**
- **Step 5**: เมื่อยอดรวมของสลิปทั้งสอง (600 + 400) เท่ากับยอดสุทธิ (1,000 บาท) กล่องแจ้งเตือนสีแดงและช่องกรอกสาเหตุจะต้อง **ไม่ปรากฏ**
- **Step 6**: สร้างคำสั่งซื้อสำเร็จ

## 🟢 TC4: การอัปเซลคำสั่งซื้อเดิม (ยอดโอนตรงตามบิล)

**Pre-condition:**
- ผู้ใช้กดปุ่ม "อัปเซล" เพื่อเข้าสู่โหมดอัปเซล และเลือกออเดอร์เป้าหมาย
- ออเดอร์เป้าหมายเป็นรูปแบบ **โอนเงิน (Transfer)**

**Steps:**
1. เพิ่มรายการสินค้าในตาราง (เช่น สินค้าราคา 500 บาท)
2. อัปโหลดสลิปใหม่
3. เลือกระบุวันที่-เวลาโอน
4. ระบุยอดเงิน **500 บาท** ให้ตรงกับยอดเพิ่มใหม่ (Upsell New Items Total)
5. สังเกตว่าจะ **ไม่มี** ข้อความแจ้งเตือนยอดไม่ตรงปรากฏ
6. กดปุ่ม "บันทึกข้อมูลอัปเซล"

**Expected Result:**
- บันทึกการอัปเซลสำเร็จ สลิปูกบันทึก

---

## 🟡 TC5: การอัปเซลคำสั่งซื้อเดิม (ยอดโอนไม่ตรงตามบิล - ต้องระบุสาเหตุ)

**Pre-condition:**
- เข้าสู่โหมดอัปเซลและเลือกออเดอร์โอนเงิน
- ยอดเพิ่มใหม่ (Upsell New Items Total) = 500 บาท

**Steps:**
1. อัปโหลดสลิปใหม่
2. เลือกระบุวันที่-เวลาโอน และระบุยอดเงินเป็น **800 บาท**
3. สังเกต UI แจ้งเตือนยอดโอนเกิน
4. กดปุ่ม "บันทึกข้อมูลอัปเซล" โดยปล่อยช่องสาเหตุว่างไว้
5. กรอกข้อมูลในช่อง "สาเหตุยอดไม่ตรงบิล" (เช่น "โอนรวมกับบิลอื่น") 
6. กดปุ่ม "บันทึกข้อมูลอัปเซล" อีกครั้ง

**Expected Result:**
- **Step 3**: ระบบแสดงกล่อง Warning และช่องกรอกสาเหตุที่ใบสลิป
- **Step 4**: ระบบจะบล็อคการบันทึกและแสดง Text Error บริเวณเหนือรายการสลิปว่า "ยอดโอนไม่ตรงกับยอดเพิ่มใหม่ กรุณาระบุสาเหตุ..."
- **Step 6**: บันทึกการอัปเซลสำเร็จ ข้อมูลถูกอัปเดตลง Backend

## 🟢 TC6: การตรวจสอบรูปแบบเบอร์โทรศัพท์ลูกค้า

**Pre-condition:**
- ผู้ใช้เข้าสู่เมนูสร้างคำสั่งซื้อปกติ และกำลังเพิ่ม/แก้ไขเบอร์ลูกค้า (ทั้งเบอร์หลักและเบอร์สำรอง)

**Steps:**
1. ทดลองกรอกเบอร์โทรที่ **ไม่ได้ขึ้นต้นด้วยเลข 0** (เช่น `891234567`)
2. สังเกตข้อความแจ้งเตือนสีแดงใต้ช่องกรอก
3. ทดลองกรอกเบอร์โทรที่ขึ้นต้นด้วย 0 แต่ความยาว **น้อยกว่า 9 หลัก** (เช่น `02123456`)
4. สังเกตข้อความแจ้งเตือนสีแดง
5. ทดลองกรอกเบอร์โทรบ้าน **9 หลัก** (เช่น `021234567`) และเบอร์มือถือ **10 หลัก** (เช่น `0891234567`)

**Expected Result:**
- **Step 2**: ระบบแสดง Error: "เบอร์โทรต้องขึ้นต้นด้วย 0"
- **Step 4**: ระบบแสดง Error: "เบอร์โทรต้องมี 9 หรือ 10 หลัก"
- **Step 5**: ระบบไม่แสดง Error สามารถผ่าน Validation ไปได้

## 🟡 TC7: การซ่อนปุ่ม "สร้างรายชื่อใหม่" ตามสิทธิ์ผู้ใช้งาน

**Pre-condition:**
- `User A`: มีสิทธิ์ (role) เป็น `Telesale` หรือ `Supervisor Telesale`
- `User B`: มีสิทธิ์เป็น Role อื่นๆ เช่น `Admin Page` (role ≠ Telesale, Supervisor Telesale)

**Steps:**
1. ล็อกอินด้วย `User A`
2. เข้าเมนู "สร้างคำสั่งซื้อ" และพิมพ์ค้นหาชื่อลูกค้าแบบมั่วๆ (เช่น "asdfgh")
3. สังเกตผลการค้นหา และลิงก์ด้านล่างผลการค้นหา
4. ล็อกอินด้วย `User B`
5. ทำขั้นตอนที่ 2 ซ้ำ

**Expected Result:**
- **User A (Step 3)**: จะ **ไม่เห็น** ปุ่ม/ข้อความ "ไม่พบลูกค้านี้ในระบบ? สร้างรายชื่อใหม่"
- **User B (Step 5)**: จะ **มองเห็น** ปุ่ม/ข้อความ "ไม่พบลูกค้านี้ในระบบ? สร้างรายชื่อใหม่" และสามารถกดเพื่อสร้างลูกค้าได้
