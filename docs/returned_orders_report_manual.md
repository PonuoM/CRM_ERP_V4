# คู่มือการใช้งานหน้ารายงานออเดอร์ตีกลับและยกเลิก (Returned & Cancelled Orders Report)

เอกสารนี้อธิบายการทำงานและโครงสร้างของหน้า UI สำหรับ **"Special Orders Report"** (รายงานออเดอร์เคสพิเศษ) ซึ่งประกอบไปด้วยระบบดูรายงานออเดอร์ตีกลับ/ยกเลิก, ระบบค้นหา/กรองข้อมูล, การดึงไฟล์เสียงอัตโนมัติ, และระบบจัดการรายละเอียดแบบเบ็ดเสร็จ (Unified Order Details Modal)

---

## 1. ภาพรวมของระบบ (Overview)

ระบบถูกออกแบบมาเพื่อให้ทีมงาน (Admin, QA, Manager) สามารถตรวจสอบออเดอร์ที่ถูก "ตีกลับ (Returned)" หรือ "ยกเลิก (Cancelled)" ได้อย่างรวดเร็ว โดยมีความสามารถหลักดังนี้:
- **แยกแท็บการทำงานชัดเจน**: สลับไปมาระหว่างออเดอร์ตีกลับและยกเลิก
- **Dashboard สรุปยอด**: แสดงยอดจำนวนออเดอร์รวมและยอดเงินรวมประจำช่วงวันที่เลือก
- **ระบบ Filter**: กรองข้อมูลตามช่วงวันที่ (Date Range), รหัสพนักงาน (User ID), และสถานะการจัดการ (All, Pending, Completed)
- **ระบบไฟล์เสียงอัจฉริยะ**: สามารถดึงลิงก์ไฟล์เสียงจากการโทรในอดีต (Google Drive URL) แบบอัตโนมัติ (Auto-match) จากตู้สาขา (PBX)
- **Unified Order Details Modal**: หน้าต่างจัดการรายละเอียดเบ็ดเสร็จในปุ่มเดียว สำหรับเขียนสรุปผลการพิจารณาและจัดการลิงก์ไฟล์เสียง

---

## 2. ระบบการกรองสถานะการจัดการ (Resolution Status Filter)

เพื่อช่วยให้ทีมงานไม่ทำงานซ้ำซ้อนและสามารถติดตามงานที่ยังค้างอยู่ได้ ระบบจึงเพิ่มสถานะ **"การจัดการ (Resolution)"** เข้ามา:

- **ทั้งหมด (All)**: แสดงทุกออเดอร์
- **รอดำเนินการ (Pending)**: ออเดอร์ที่ยังไม่ได้กด "ทำเครื่องหมายว่าเสร็จ" (พื้นหลังปกติ)
- **จัดการเรียบร้อยแล้ว (Completed)**: ออเดอร์ที่พิจารณาและเขียนสรุปเสร็จแล้ว (พื้นหลังตารางจะถูกปรับเป็นสีเทาจาง (Fade-out) และตัวอักษรจางลง เพื่อลดสิ่งรบกวนสายตา)

ผู้ใช้สามารถสลับสถานะของแต่ละออเดอร์ได้โดยการกดปุ่ม **ทำเครื่องหมายว่าเสร็จ** / **สรุปแล้ว (Completed)** ที่คอลัมน์ขวาสุดของตาราง

---

## 3. หน้าต่างจัดการรายละเอียดแบบเบ็ดเสร็จ (Unified Order Details Modal)

*(Best Practice UX/UI และ Database Transaction)*

เมื่อคลิกปุ่ม **"จัดการรายละเอียด"** ระบบจะเปิด Modal กลางขึ้นมา ซึ่งรวมเอาฟังก์ชันการจัดการ 3 อย่างไว้ด้วยกัน เพื่อให้ผู้ใช้ทำงานจบได้ในหน้าต่างเดียว:

### 3.1 การสรุปเคสคำสั่งซื้อ (Order Summary)
- มีช่อง Textarea ขนาดใหญ่สำหรับใส่ข้อความสรุปการพิจารณา เช่น "ลูกค้าเปลี่ยนใจกะทันหัน", "ติดต่อลูกค้าไม่ได้เกิน 3 วัน"

### 3.2 การจัดการลิงก์ไฟล์เสียง (Audio Links Management)
- **ไฟล์เสียงเดิม**: จะแสดงรายการลิงก์ไฟล์เสียงที่เคยแนบไว้ สามารถแก้ลิงก์ (URL), แก้ไขวันเวลาที่โทร, แก้ไขหมายเหตุของลิงก์นั้นๆ หรือกดลบ (Trash icon) ได้
- **เพิ่มไฟล์เสียงใหม่**: หากกดปุ่ม "เพิ่มไฟล์เสียงใหม่" ระบบจะเปิดฟอร์มย่อยให้กรอกลิงก์ Google Drive ใหม่ (สามารถเพิ่มได้หลายลิงก์พร้อมกันก่อนกดเซฟ)

### 3.3 กลไกความปลอดภัยและประสิทธิภาพ
- **Dirty State Alert**: หากผู้ใช้เผลอกดปิด Modal ในขณะที่พิมพ์ข้อมูลค้างไว้ (ยังไม่ได้กดบันทึก) ระบบจะแจ้งเตือน (Confirm Close) ป้องกันข้อมูลที่อุตส่าห์พิมพ์หายไป
- **Data Delta Sync**: เมื่อกดบันทึก ระบบจะส่งต่อเฉพาะข้อมูลที่มีการเปลี่ยนแปลงไปยัง API (เฉพาะลิงก์ที่ถูกลบ, ถูกแก้, เพิ่มใหม่) 
- **Database Transaction (Backend)**: ในไฟล์ `ReturnedOrdersReportService.php` การบันทึกสรุปและไฟล์เสียงทั้งหมดจะถูกมัดรวมด้วยคำสั่ง `beginTransaction()` หากเกิดข้อผิดพลาดใดๆ ขึ้น ระบบจะ `rollBack()` ทันที เพื่อป้องกันฐานข้อมูลพัง (Data Inconsistency)

## 4. โครงสร้างไฟล์ (File Structure)

- **UI Page**: `pages/ReturnedOrdersReportPage.tsx`
  - ควบคุม State ทั้งหมด (วันที่, แท็บ, ฟิลเตอร์, เปิด/ปิด Modal)
- **Modal Component**: `components/ReturnedOrdersReport/OrderDetailsModal.tsx`
  - หน้าต่างจัดการข้อมูลทั้งหมด จัดการเรื่อง Dirty State และ Differential Payload
- **Backend API**: `api/index.php` 
  - จุดรับ Request (`POST returned_orders_report/update-details`, `toggle-completed`)
- **Backend Service**: `api/Services/ReturnedOrdersReportService.php`
  - รันคำสั่ง SQL ภายใต้ Transaction เพื่ออัปเดตตาราง `orders` และ `order_audio_links`
- **Database Migration**: `api/migrations/045_add_admin_resolution_completed_to_orders.sql`
  - โครงสร้างคอลัมน์ `admin_resolution_completed` สำหรับเช็คสถานะการจัดการ
