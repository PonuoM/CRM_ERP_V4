# 💰 คู่มือระบบอนุมัติการเงินและการกระทบยอด (Finance Approval System)

ไฟล์หน้าจอหลัก: `pages/FinanceApprovalPage.tsx`

ระบบ **Finance Approval** เป็นศูนย์กลางสำหรับแผนกการเงิน (Finance / Accounting) ในการตรวจสอบและยืนยันยอดเงินที่ลูกค้าชำระเข้ามา หรือยอดเงินที่บริษัทขนส่งโอนให้ (COD) โดยระบบจะทำการ "กระทบยอด" (Reconciliation) ระหว่างยอดเงินในบัญชีธนาคารจริงๆ (Statement) กับรายการคำสั่งซื้อ (Orders) ในระบบ

---

## 📌 ภาพรวมฟีเจอร์หลัก (Key Features)

ระบบนี้แบ่งการทำงานหลักออกเป็น 3 แท็บ (Tabs) ดังนี้:

### 1. แท็บ "โอนเงิน" (Transfers / Slip Matching)
ใช้สำหรับตรวจสอบออเดอร์ที่ลูกค้าชำระเงินด้วยวิธี **โอนเงิน (Bank Transfer)**
- **ค้นหา Statement:** แผนกการเงินสามารถเลือกบัญชีธนาคาร และระบุช่วงวันที่เพื่อดึงรายการเดินบัญชี (Statement Logs) ที่อิมพอร์ตเข้ามาในระบบ
- **ระบบ Auto Match:** ระบบจะพยายามจับคู่อัตโนมัติ (Auto Reconciliation) ระหว่างยอดเงินจากธนาคาร กับยอดรวมของออเดอร์ (หรือสลิป) โดยมีเงื่อนไขคือ:
  - ยอดเงินต้องตรงกันเป๊ะ หรือต่างกันไม่เกิน 1 บาท
  - เวลาโอนต้องต่างกันไม่เกิน 3 นาที
  - เลขบัญชีธนาคารต้องตรงกัน
- **ระบบแนะนำ (Candidates & Scoring):** หากระบบจับคู่อัตโนมัติไม่ได้ ระบบจะแสดงปุ่ม "ค้นหา" เพื่อโชว์รายการออเดอร์ที่เป็นไปได้ (Candidates) พร้อมให้คะแนนความใกล้เคียง (Score) โดยจัดกลุ่มเป็น:
  - `Exact`: ยอดตรงเป๊ะ เวลาคลาดเคลื่อนไม่เกิน 5 นาที
  - `Close`: ยอดคลาดเคลื่อนเล็กน้อย เวลาคลาดเคลื่อนไม่เกิน 20 นาที
  - `Suggestion`: ยอดคลาดเคลื่อนหลักร้อย หรือเวลาคลาดเคลื่อนเกิน 30 นาที
- **การยืนยัน:** เมื่อกดบันทึก ออเดอร์ที่ถูกจับคู่จะถูกเปลี่ยนสถานะเป็น "ชำระเงินแล้ว (Paid)" 

### 2. แท็บ "เก็บปลายทาง" (Pay After / COD Reconciliation)
ใช้สำหรับตรวจสอบยอดเงินโอนเข้าจาก **บริษัทขนส่ง (COD)** 
- **เอกสาร COD (COD Documents):** ดึงรายการเอกสารสรุปยอด COD (ที่อิมพอร์ตเข้ามาจากไฟล์ Excel ของขนส่ง) ซึ่งสถานะยังเป็น `pending` หรือ `unmatched`
- **จับคู่ยอดรวม (Batch Matching):** แผนกการเงินจะต้องจับคู่ยอดเงินรวมของเอกสาร COD (Total Input Amount) ให้ตรงกับยอด Statement ของธนาคารแบบ 1-to-1
- **รายละเอียด Tracking:** สามารถกดดูรายละเอียดภายในเอกสาร COD ว่ามีออเดอร์ไหนบ้างที่ขนส่งโอนเงินมาให้ และมี Tracking Number อะไรบ้าง (ระบบจะเทียบยอดเงิน COD จากขนส่ง กับยอดออเดอร์จริงในระบบ หากไม่ตรงกันจะแสดงแถบสีแดงให้เห็นส่วนต่าง)
- **การยืนยัน:** เมื่อกระทบยอดเอกสาร COD สำเร็จ ระบบจะบันทึกสถานะเอกสารเป็น Verified และอัปเดตสถานะออเดอร์ทั้งหมดในเอกสารนั้นเป็นชำระเงินแล้ว

### 3. แท็บ "ตรวจสอบสลิปแนบ" (Slips - *Legacy/Hidden*)
- สำหรับการอนุมัติสลิปทีละใบแบบ Manual (ปัจจุบันระบบมุ่งเน้นไปที่การทำ Auto Match แบบ Statement Reconciliation เป็นหลัก จึงอาจถูกปิดการใช้งานด้วย Flag `SHOW_APPROVE_SLIP = false`)

## ⚙️ การทำงานของระบบจับคู่ (Reconciliation Logic)

1. **`secondsDiff(a, b)`**: ฟังก์ชันคำนวณผลต่างของเวลา (เป็นวินาที) ระหว่างเวลาโอนเงินใน Statement กับเวลาโอนเงินที่แจ้งในสลิป
2. **`buildCandidates`**: ฟังก์ชันคำนวณคะแนนความใกล้เคียง (Score):
   ```javascript
   const score = 
      (amountDiff * 100) +          // ยอดเงินต่างกัน หักคะแนนเยอะ
      Math.min(timeDiff, 1800) +    // เวลาต่างกัน หักคะแนนตามวินาที (สูงสุด 30 นาที)
      (bankPenalty * 10000) +       // บัญชีธนาคารไม่ตรงกัน หัก 10,000 คะแนน
      (alreadyUsed * 5000);         // ออเดอร์นี้ถูกใช้ไปแล้ว หัก 5,000 คะแนน
   ```
   *หมายเหตุ: คะแนนยิ่งน้อย ยิ่งหมายถึงมีความใกล้เคียงมาก (Best Match)*

## 🛡️ Best Practices & UX

- **Bulk Save:** แผนกการเงินสามารถกดยืนยัน (Check) หลายๆ รายการพร้อมกัน แล้วกดปุ่ม "บันทึก" เพียงครั้งเดียว (Batch Request) ช่วยลดเวลาทำงานได้มาก
- **Safety Checks:** ระบบจะป้องกันไม่ให้อนุมัติซ้ำซ้อน หากมี Statement Log ไหนถูกจับคู่ไปแล้ว จะไม่สามารถนำมาจับคู่กับออเดอร์อื่นได้อีก
- **Partial Payment Validation:** มีฟังก์ชันคอยคำนวณ `remaining` (ยอดคงเหลือ) เพื่อให้ทราบว่าหากลูกค้าชำระเงินมาบางส่วน ออเดอร์นี้ยังขาดเงินอีกเท่าไหร่
