# ระบบป้ายกำกับลูกค้า (Customer Tag System)

## ภาพรวม
ระบบ Tag ออกแบบมาเพื่อให้พนักงานขาย (Telesale) หรือ Admin สามารถจัดกลุ่มและทำสัญลักษณ์ให้ลูกค้าแต่ละคนได้ เพื่อความสะดวกในการติดตามงาน โดยระบบรองรับ Tag 2 ประเภทหลัก ได้แก่ **SYSTEM Tags (ป้ายกำกับส่วนกลาง)** และ **USER Tags (ป้ายกำกับส่วนตัว)**

---

## Database Schema (ตารางที่เกี่ยวข้อง)

ระบบประกอบด้วย 3 ตารางหลัก:

### 1. `tags` (ข้อมูล Tag หลัก)
เก็บข้อมูลพื้นฐานของแต่ละป้ายกำกับ
- `id`: รหัสป้ายกำกับ
- `name`: ชื่อป้ายกำกับ (เช่น "ลูกค้า VIP", "ตามเงิน")
- `type`: ประเภทของป้ายกำกับ มีค่าเป็น `SYSTEM` หรือ `USER`
- `color`: รหัสสี (Hex code) สำหรับแสดงผลบน UI

### 2. `user_tags` (สิทธิ์ความเป็นเจ้าของ - Ownership)
เก็บข้อมูลว่า Tag นั้นใครเป็นเจ้าของ (สำหรับ Personal Tag)
- `user_id`: รหัสผู้ใช้งาน (Telesale)
- `tag_id`: รหัสป้ายกำกับ

### 3. `customer_tags` (การแปะป้ายให้ลูกค้า - Assignment)
เก็บข้อมูลว่าลูกค้าคนไหน ถูกแปะป้ายอะไรเอาไว้บ้าง
- `customer_id`: รหัสลูกค้า
- `tag_id`: รหัสป้ายกำกับ

---

## กฎการใช้งานระบบ Tag (Business Logic)

### 1. โควตา Personal Tag (USER Type)
- พนักงานแต่ละคนสามารถสร้างป้ายกำกับส่วนตัว (USER Tag) ได้ **สูงสุด 10 ป้าย**
- หากพยายามสร้างป้ายที่ 11 ระบบจะคืนค่า Error `TAG_LIMIT_REACHED` (400) กลับไป

### 2. การมองเห็น (Visibility)
ระบบใช้กฎ **"สิทธิ์การมองเห็นตาม Session ผู้ใช้งานปัจจุบัน"** (อ้างอิงรหัสจากคนที่ล็อกอินเข้ามา `$authUserId`) 
เพื่อให้แน่ใจว่าเรื่องส่วนตัวของใคร ก็จะเห็นเฉพาะของคนนั้น โดยมีเงื่อนไขดังนี้:
1. **เห็น SYSTEM Tag เสมอ:** เพราะถือเป็นของส่วนกลาง (`user_tags.user_id IS NULL`)
2. **เห็น USER Tag ของตัวเองเท่านั้น:** ป้ายส่วนตัวที่พนักงานคนอื่นสร้างไว้ พนักงานคนอื่นจะไม่มีสิทธิ์มองเห็น (`user_tags.user_id = $authUserId`)
3. **ติดตัวลูกค้าไปเสมอ:** ไม่ว่าลูกค้าคนนั้นจะถูกโอนย้าย (`assigned_to`) ไปให้ใครก็ตาม Tag ส่วนตัวของคุณที่เคยแปะไว้ก็จะยังคงอยู่ และคุณจะยังมองเห็น Tag นั้นได้เสมอ

**SQL Logic ที่ใช้ในการ Query (ตัวอย่างจาก `handle_customers`):**
```sql
SELECT t.* 
FROM tags t 
JOIN customer_tags ct ON ct.tag_id = t.id 
LEFT JOIN user_tags ut ON ut.tag_id = t.id 
WHERE ct.customer_id = ? 
AND (ut.user_id IS NULL OR ut.user_id = ?) -- ? ตัวหลังคือรหัสคนที่ล็อกอิน ($authUserId)
```

### 3. ระบบแยก Tag ที่ซ้ำซ้อน (Tag Separation)
ในอดีตอาจมีการแชร์ ID ของ Personal Tag ตัวเดียวกันให้กับผู้ใช้หลายคน (1 Tag > N Users) ทำให้เกิดบั๊กเห็น Tag ข้ามคน ปัจจุบันได้มีการแยก Tag แบบ 1 ต่อ 1 แล้ว สคริปต์สำหรับการ Migration คือ `run_tag_split.php` (ดูโค้ดเก่าได้ใน History)

## API Endpoints (`api/index.php`)

### `handle_tags` (จัดการข้อมูล Tag)
- **GET `?action=tags`:** 
  - ส่ง `type=SYSTEM` เพื่อขอ Tag ระบบทั้งหมด
  - ส่ง `type=USER&userId=X` เพื่อขอ Tag ส่วนตัวของพนักงานรหัส X
- **POST `?action=tags`:** สร้าง Tag ใหม่ หากเป็นประเภท `USER` ระบบจะเพิ่มแถวในตาราง `user_tags` อัตโนมัติ (และเช็คโควตา 10 ป้าย)
- **PUT `?action=tags&id=X`:** แก้ไขชื่อหรือสีของ Tag
- **DELETE `?action=tags&id=X`:** ลบ Tag ทิ้ง (และลบความสัมพันธ์ใน `customer_tags`, `user_tags` ด้วยแบบ CASCADE)

### `handle_customer_tags` (จัดการการแปะป้ายให้ลูกค้า)
- **POST `?action=customer_tags`:** แปะป้ายให้ลูกค้า (ส่ง `customerId` และ `tagId`)
- **DELETE `?action=customer_tags`:** ลบป้ายออกจากลูกค้า (ส่ง `customerId` และ `tagId`)

## Frontend Integration (การแสดงผลบน UI)

ระบบ Tag ถูกนำไปใช้งานในหลายหน้าจอหลักของระบบ ได้แก่:

### 1. หน้า Dashboard / หน้าแจกงาน (`TelesaleDashboardV2.tsx`, `DistributionV2.tsx`)
- ในตารางรายชื่อลูกค้า จะมีป้าย Tag สีสันต่างๆ ปรากฏอยู่ใต้ชื่อลูกค้า หรือในคอลัมน์ชื่อลูกค้า
- หากลูกค้ามี Tag จำนวนมาก ระบบจะแสดง 2 Tag แรก และโชว์เป็นตัวเลข `+N` (เช่น `+2`) ให้เอาเมาส์ไปชี้ (Hover) เพื่อดูชื่อ Tag ที่ซ่อนอยู่ได้

### 2. หน้าจัดการลูกค้า / หน้ารายงาน (`CustomerTable.tsx`)
- คอมโพเนนต์ตารางลูกค้ากลางที่ใช้ทั้งในหน้า Reports, Admin Dashboard และ Telesale Dashboard จะรองรับการแสดงผล Tag ติดตัวลูกค้ามาด้วยเสมอ

### 3. หน้ารายละเอียดลูกค้า (`CustomerDetailPage.tsx`)
- ในหน้าโปรไฟล์ลูกค้า จะมีส่วนของ **"ป้ายกำกับลูกค้า"** โดยเฉพาะ แสดงรายการ Tag ทั้งหมดที่ลูกค้าคนนี้มี

### 4. หน้าต่างจัดการ Tag (`TagManagementModal.tsx`)
- เป็น Modal (Pop-up) กลางของระบบ (ถูกเรียกใช้ที่ `App.tsx`)
- **การเพิ่ม/ลบ:** เมื่อกดปุ่มแก้ไข Tag ระบบจะเปิด Modal นี้ขึ้นมาให้ติ๊กเลือก Tag หรือกดค้นหา/สร้าง Tag ใหม่ได้
- ระบบจะดึงเอาทั้ง `SYSTEM Tags` และ `USER Tags` ของพนักงานคนนั้นมาให้เลือก หากพิมพ์ค้นหาชื่อ Tag แล้วไม่เจอ จะมีปุ่มให้กดสร้าง **"ป้ายกำกับส่วนตัวใหม่"** ได้ทันที (ซึ่งจะวิ่งไปเข้าเงื่อนไขโควตา 10 ป้าย)
