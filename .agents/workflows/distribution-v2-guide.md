# 📦 Distribution V2 คืออะไร? (อธิบายแบบง่ายๆ)

ลองนึกภาพว่าเราเป็น **หัวหน้าทีม (Supervisor)** ที่ต้อง **แจกรายชื่อลูกค้า** ให้พนักงานไปโทรขาย หน้า Distribution V2 นี่แหละคือ **"หน้าจอแจกงาน"** ที่ใช้ทำเรื่องนี้!

---

## 🎯 มันคืออะไรกันแน่?

Distribution V2 เปรียบเหมือน **"ห้องแบ่งงาน"** ที่หัวหน้าเอาลูกค้าจาก "กองกลาง" มาแจกให้พนักงานแต่ละคน

**ตัวอย่างง่ายๆ:**
- เหมือนกับมีกล่องลูกอมใหญ่ (กองกลาง)
- หัวหน้าแบ่งลูกอมให้ลูกน้องคนละเท่าๆ กัน
- ลูกน้องแต่ละคนก็เอาไปขาย

---

## 🗂️ ถังแจกงาน (Distribution Baskets) มีอะไรบ้าง?

นี่คือ **"กองกลาง"** ที่มีลูกค้ารอแจก:

| ถัง | ความหมาย | แจกเข้าถังไหนของ Telesale |
|-----|----------|---------------------------|
| 📦 **Upsell** | ลูกค้าที่สั่งของล่าสุด | ถัง Upsell |
| ⏳ **พักรอแจก** | ลูกค้าที่เพิ่งถูกดึงคืนมา (รอ 30 วัน) | *ปกติแจกไม่ได้ แต่กดแจกไปถัง "หาคนดูแลใหม่" ได้* |
| 📦 **New Customer** | ลูกค้าใหม่ | ถังลูกค้าใหม่ |
| 📦 **ค้นหาเจ้าของใหม่** | ลูกค้าที่รอคนดูแล | ถัง 1-2 เดือน |
| 📦 **รอจับคู่** | ลูกค้าที่รอจัดสรร | ถัง Last Chance |
| 📦 **6-12 เดือน** | ไม่สั่งมา 6-12 เดือน | ถัง 6-12 เดือน |
| 📦 **1-3 ปี** | ไม่สั่งมา 1-3 ปี | ถัง 1-3 ปี |
| 📦 **Ancient** | ไม่สั่งมานานมาก | ถัง Ancient |

```
พิมพ์จำนวนลูกค้าที่ต้องการแจก
```

```
✓ ติ๊กเลือกพนักงานที่จะแจกให้
  - เลือกได้หลายคน
```

### ขั้นตอนที่ 4: ดูตัวอย่างก่อนแจก

## 🔙 การจัดการลูกค้าของพนักงาน (ดึงคืน / โอนย้าย) [NEW BULK ACTION]

บางทีต้อง **เอาลูกค้าคืน** จากพนักงาน หรือ **โอนให้คนอื่น** เช่น:
- พนักงานลาออก
- ต้องการปรับสมดุลงาน
- พนักงานดองลูกค้าไว้โดยไม่มีการนัดหมาย

ระบบได้เปลี่ยนหน้าต่างใหม่เป็น **"Bulk Action Modal"** ที่จัดการได้ง่ายขึ้นและทำได้หลายถังพร้อมกัน:

### การทำงานของ Bulk Action Modal
1. **กดปุ่ม "ดึงคืน / โอน"** (สัญลักษณ์ Action) ที่แถวของพนักงาน
2. ระบบจะเปิดหน้าต่างขึ้นมา โดยแบ่งเป็น 2 ส่วนหลัก:
   - **เลือกประเภทการทำงาน:** ดึงคืน (กลับถังกลาง) หรือ โอน (ให้พนักงานอื่น)
   - **ตัวเลือกเสริม (เฉพาะดึงคืน):** 
     - ดึงคืนทั้งหมด
     - เฉพาะไม่มีนัดหมาย
     - เฉพาะคนที่โทรแล้วไม่นัดหมาย
3. **เลือกลูกค้าที่จะดำเนินการ (Select Baskets):**
   - มีปุ่ม `☑ เลือกทั้งหมด (Select All)`
   - สามารถพิมพ์ **ระบุจำนวนสูงสุด (Limit)** ที่ต้องการดึงจากแต่ละถังได้
   - ระบบจะ **Smart Cap (คำนวณอัตโนมัติ)** ให้ว่าถังไหนมีสูงสุดกี่คน หากพิมพ์เกิน ระบบจะดึงแค่เท่าที่มีให้โดยไม่ Error
4. **กดปุ่ม "ยืนยันการดึงคืน / โอน"**

### 🌟 หน้าต่างสรุปผลลัพธ์ (Result Modal)
เมื่อดำเนินการเสร็จสิ้น ระบบจะไม่แสดงแค่ข้อความเล็กๆ อีกต่อไป แต่จะเด้ง **หน้าต่าง Modal สรุปผลลัพธ์แบบละเอียด**:
- **ยอดรวม:** แสดงจำนวนลูกค้าทั้งหมดที่ถูกจัดการสำเร็จ
- **รายละเอียดแต่ละถัง:** แสดงให้เห็นชัดเจนว่า
  - ถังอะไร ถูกดึงออกมากี่คน
  - **จาก:** พนักงานคนไหน ➔ **ไปที่:** พนักงานคนไหน (หรือ ถังกลาง)

## 🔄 Manual Reset (ล้างรอบด้วยตนเอง) [NEW]

ล้างประวัติการถือครอง (`customer_assign_check`) เพื่อแจกซ้ำ agent เดิมได้อีกครั้ง

### วิธีใช้งาน:

1. **กดปุ่ม "Manual Reset"** มุมขวาบน
2. **เลือกรอบ** จาก dropdown (เช่น "รอบที่ 1 ลูกค้า 67,329 รายชื่อ") → กด **ค้นหา**
3. **ใช้ Filter:**
   - 🔍 **ค้นหา**: พิมพ์ชื่อ/เบอร์โทร/รหัสลูกค้า → Enter หรือกด "กรอง"
   - 👤 **กรอง Agent**: เลือก agent → แสดงเฉพาะลูกค้าที่เคยแจกให้ agent นั้น
4. **ตาราง** แสดง: รหัส, ชื่อ-นามสกุล, เบอร์โทร, เคยแจกให้ (agent names), จำนวนครั้ง
5. **Reset**:
   - ✅ **Reset ที่เลือก**: ติ๊กเลือกรายชื่อ → กดยืนยัน
   - ⚠️ **Reset ทั้งหมด**: ล้างทั้งหมดตามเงื่อนไข (ต้องยืนยัน 2 ชั้น)

### API Endpoints (`Distribution/reset.php`):
| Action | คำอธิบาย |
|--------|----------|
| `get_reset_summary` | ดึงสรุปจำนวนตามรอบ |
| `get_candidates` | ดึงรายชื่อ (รองรับ `search`, `agent_id`) |
| `get_agents_in_checks` | ดึง agent list สำหรับ dropdown |
| `manual_reset` | ลบ assign check + เพิ่ม current_round |

## 🚫 ตะกร้าลูกค้าบล็อค [NEW]

ตะกร้า **"ลูกค้าบล็อค"** เป็นตะกร้าพิเศษ — **กดแจกไม่ได้** แต่เมื่อกดจะเปิด Modal แสดงรายชื่อ:

### วิธีใช้งาน:

1. **กดที่การ์ด 🚫 ลูกค้าบล็อค** (ตัวอักษรสีแดง)
   → เปิด Modal แสดงรายชื่อลูกค้าที่ถูกบล็อค

2. **ค้นหา/กรอง**: พิมพ์ชื่อ/เบอร์ หรือเลือกกรองตามสาเหตุ

3. **เลือกรายชื่อ**: ติ๊ก checkbox (เลือกได้หลายคน / Select All)

4. **เลือกตะกร้าปลายทาง**: dropdown แสดงเฉพาะตะกร้า distribution

5. **กด "✅ ปลดบล็อค"**:
   - `customer_blocks.active = 0`
   - `customers.is_blocked = 0`
   - `customers.current_basket_key = {ตะกร้าที่เลือก}`

### 🔍 ตรวจสอบ Sync:

ปุ่ม **"ตรวจสอบ Sync"** ที่ header ใช้เช็คว่ามีลูกค้าที่ `is_blocked=1` แต่ยังไม่ได้อยู่ตะกร้าบล็อค (basket 55) หรือไม่:
- ถ้าพบ → แสดงรายชื่อ + ตะกร้าปัจจุบัน + ประเภท (target_page)
- กด **"📦 ย้ายทั้งหมดเข้าตะกร้าบล็อค"** เพื่อ fix

## 🧠 Smart Allocation (ระบบแจกอัจฉริยะ)

ระบบมี **"ความฉลาด"** ในการแจกลูกค้า:

### ✅ สิ่งที่ระบบทำให้อัตโนมัติ:

1. **ดึงข้อมูลซ้ำจาก DB ก่อนแจก**
   - เรียก API `get_assign_checks` เพื่อดึงข้อมูลจากตาราง `customer_assign_check`
   - รู้ว่าลูกค้าแต่ละคนเคยอยู่กับพนักงานคนไหนบ้าง

2. **จัดลูกค้ายากสุดก่อน (Most Constrained First)**
   - ลูกค้าที่เคยอยู่กับ agent หลายคน → มีตัวเลือกน้อย → ต้องจัดก่อน

3. **ไม่แจกซ้ำคนเดิม + แจกเท่าๆ กัน**
   - เลือก agent ที่: ไม่ซ้ำ + ได้ลูกค้าน้อยสุดในรอบนี้
   - *ตัวอย่าง*: เลือก 4 คน แจก 100 คน = คนละ 25 คน

4. **Fallback (ถ้าซ้ำทุก agent)**
   - ถ้าลูกค้าเคยอยู่กับทุก agent แล้ว → ยอมให้ agent ที่ได้น้อยสุด

### ⚠️ คำเตือนที่อาจเจอ:

| ข้อความ | ความหมาย |
|---------|----------|
| "บางพนักงานได้น้อยกว่ากำหนด" | มีลูกค้าซ้ำ ระบบข้ามไป |
| "ไม่มีลูกค้าพร้อมแจก" | ถังว่างแล้ว |
| "กรุณาเลือกพนักงาน" | ยังไม่ได้ติ๊กเลือกใคร |

## ❓ FAQ (คำถามที่พบบ่อย)

**Q: ทำไมบางถังกดดึงคืนไม่ได้?**
> A: ถัง Upsell ดึงคืนไม่ได้ และถังที่ไม่มี linked basket ก็ดึงไม่ได้

**Q: แจกไปแล้วยกเลิกได้ไหม?**
> A: ได้! ใช้ปุ่ม "ดึงคืน" เพื่อเอาลูกค้ากลับมา

**Q: ทำไมพนักงานได้ไม่เท่ากัน?**
> A: ระบบจะข้ามลูกค้าที่เคยแจกให้คนเดิมแล้ว (ไม่แจกซ้ำ) และเลือก agent ที่ได้น้อยสุด ถ้าซ้ำทุกคนจะ fallback ให้ agent ที่ได้น้อยสุด

**Q: ลูกค้าที่ดึงคืนไปไหน?**
> A: กลับไปที่ถังแจกงานที่ linked กับถังนั้น

**Q: ตะกร้าลูกค้าบล็อคกดแจกทำไมไม่ได้?**
> A: ตะกร้าบล็อคจะเปิด modal ให้เลือกปลดบล็อคและย้ายไปตะกร้าอื่นแทน ไม่สามารถแจกตรงได้

**Q: ตัวเลขบนการ์ดบล็อคมาจากไหน?**
> A: ดึงจาก `get_blocked_customers.php` (customer_blocks.active=1 + is_blocked=1) ไม่ใช่ basket_customers ซึ่งกรอง is_blocked ออก

**Q: ถัง "พักรอแจก" ทำไมถึงแจกไม่ได้?**
> A: ตาม Flow ปกติ ถังนี้มีไว้พักลูกค้า 30 วันก่อนจะไหลไปถังใหม่ แต่ถ้ามีความจำเป็นเร่งด่วน คุณสามารถกดที่การ์ด "พักรอแจก" แล้วคลิกปุ่ม **"ยืนยันที่จะแจก"** ระบบจะเปิดให้คุณแจกลูกค้าในถังนี้ได้ โดยจะถูกบังคับส่งเข้าถัง **"หาคนดูแลใหม่" (Dashboard)** ของ Telesale โดยอัตโนมัติ
