# ⚙️ คู่มืออธิบายการทำงานหน้าตั้งค่าตะกร้า (Basket Settings Page)

**File Path:** `pages/BasketSettingsPage.tsx`
**API Endpoint:** `api/basket_config.php` และ `api/Database/db_manager.php` (สำหรับ Audit)

หน้านี้เป็นศูนย์กลาง (Central Hub) ของผู้ดูแลระบบ (Admin) สำหรับกำหนด **"กฎการไหลของลูกค้า" (Customer Routing & Transition Rules)** และใช้สำหรับ **ตรวจสอบความถูกต้อง (Data Integrity Audit)** ของฐานข้อมูลลูกค้า

---

## 📑 โครงสร้างหลัก (Tab Menu)

หน้าเว็บถูกแบ่งออกเป็น 7 ส่วนหลักตามลักษณะการทำงาน:

### 1. 📦 Dashboard V2 (ถังที่แสดงบนหน้า Dashboard ของ Telesale)
จัดการตะกร้าส่วนตัวที่พนักงาน Telesale จะเห็นบนหน้าจอของตัวเอง (เช่น ถัง 38 ลูกค้าใหม่, 39 รอตัดสินใจ)
- **กำหนดเงื่อนไขการเข้าถัง:** สามารถกำหนดช่วงจำนวนออเดอร์, วันที่ตั้งแต่ซื้อครั้งแรก/ล่าสุด
- **กฎเมื่อขายได้ (Sale Success):** ย้ายลูกค้าไปถังไหนต่อ (เช่น ไปถัง Follow-up หรือ ซื้อซ้ำ)
- **กฎเมื่อขายไม่ได้ / หมดเวลา (Fail / Timeout):** 
  - กำหนดเวลา "ไม่ขายภายใน (วัน)" (Fail after days)
  - กำหนดตะกร้าปลายทางหากพ้นระยะเวลา
  - **✨ [ใหม่] ขยายเวลาถือครองด้วยนัดหมาย:** สามารถยืดเวลาให้อัตโนมัติ หากลูกค้ามีการนัดหมายในระบบ (Extend by Appointments) โดยกำหนดค่าเวลาที่บวกเพิ่มต่อ 1 นัด และมีเพดานวันสูงสุด
  - **✨ [ใหม่] ขยายเวลาถือครองด้วยยอดขาย (Sales Reward):** เพิ่มวันดูแลลูกค้าอัตโนมัติหากสามารถทำยอดขายสะสม (เฉพาะ Preparing, Shipping, Delivered) ในช่วงที่อยู่ในตะกร้าได้ถึงเกณฑ์ สามารถบวกทบกับนัดหมายได้ภายใต้เพดานเวลาสูงสุด

### 2. 👥 หน้าแจกงาน (Distribution)
จัดการตะกร้ากองกลางที่ใช้สำหรับรอแจกงานให้ Telesale (เช่น ถัง 52 แจกฟรีรายวัน, ถังแจกตามยอด)
- **พักรายชื่อก่อนแจกซ้ำ (Hold Days):** กำหนดว่าลูกค้ารายนี้ต้องถูกพักกี่วัน ก่อนที่จะถูกนำมาแจกซ้ำได้
- **มีวนซ้ำ (Distribution Loop):** สามารถตั้งค่าให้ลูกค้าหมุนเวียนเปลี่ยนมือได้ โดยกำหนดจำนวนรอบสูงสุด (Max Distribution) ว่าหมุนกี่รอบถึงจะไปตะกร้าสุสาน
- **เชื่อมโยงกับถัง (Linked Basket):** ผูกตะกร้าหน้าแจกให้ตรงกับตะกร้าหน้า Dashboard V2 เพื่อสะท้อนข้อมูลชุดเดียวกัน

### 3. 🔄 คืน Pool (Return Pool)
ตั้งค่ากฎสำหรับการคืนรายชื่อลูกค้ากลับเข้ากองกลางอัตโนมัติ 
- เช่น "หากไม่โทรหาภายใน X วัน ให้ริบคืน" เพื่อรักษาความสดใหม่ของการติดตามลูกค้า

### 4. 🛡️ ตรวจสอบถัง (Reevaluate)
เครื่องมือ Audit สำหรับกวาดล้าง **"ลูกค้าที่อยู่ผิดถัง"** ให้กลับไปอยู่ถังที่ถูกต้อง
- **กลไกการสแกน:** ระบบจะเช็คประวัติการสั่งซื้อ และวันที่สั่งล่าสุด เทียบกับ `basket_config` 
- **การแก้ไข (Fix):** มีปุ่มกดย้ายลูกค้ากลับไปถังที่ถูกต้องให้แบบรวดเดียว (Batch Fix) พร้อมลงบันทึก Transition Log อัตโนมัติ

### 5. 👤 ลูกค้าหลงถัง (Misassigned)
สแกนหาลูกค้าที่ตกหล่นอยู่ใน "ตะกร้าส่วนตัว" ของ Telesale แต่ความจริงแล้วพนักงานคนนั้น **ไม่ใช่คนดูแลตัวจริง**
- ตรวจสอบจาก Field `assigned_to` ว่าตรงกับเจ้าของปัจจุบันหรือไม่ ป้องกันการกั๊กลูกค้าที่หลุดกระบวนการ

### 6. ⚠️ ค้าง Upsell (Upsell Stuck)
เครื่องมือตรวจสอบหาลูกค้าที่อยู่ใน "ถัง 51 (ส่งยอด-รออัพเซลล์)" แต่ปัจจุบัน **ไม่มีคำสั่งซื้อที่ติดสถานะ Pending แล้ว**
- ปกติถัง Upsell จะถือครองลูกค้าไว้ชั่วคราวระหว่างรอตรวจสอบสลิป หากสลิปผ่าน/ยกเลิก ลูกค้าควรกลับถังเดิม ระบบนี้จะช่วยสแกนคนที่ค้างอยู่และดึงกลับถังปกติให้

### 7. 📊 ตรวจสอบยอดขาย (Stats Audit)
เครื่องมือ Audit ความสมบูรณ์ของ Data เชิงสถิติ (Customer Stats)
- เปรียบเทียบข้อมูลสรุปในตาราง `customers` (เช่น `total_purchases`, `last_order_date`) เทียบกับรายการสั่งซื้อจริงในตาราง `orders`
- **การแก้ปัญหา:** สามารถกดปุ่มเพื่อสั่งคำนวณสถิติใหม่ (Recalculate) ให้กลับมาตรงกับความจริง 100%

## 🛠️ โครงสร้าง Form แก้ไขตะกร้า (Edit Modal)

เมื่อกด "แก้ไข" ตะกร้าใดๆ ระบบจะแสดง Form `BasketConfig` ซึ่งเชื่อมโยงกับ State และเชื่อมไปที่ API โดยมี Field สำคัญๆ ดังนี้:

- `basket_key`: รหัสเฉพาะ (ห้ามซ้ำ) เป็นแกนหลักที่ใช้ค้นหาในระบบ
- `basket_name`: ชื่อที่แสดงผลให้คนอ่าน
- **Order Tracking:** `min_order_count`, `max_order_count`, `min_days_since_order`, `max_days_since_order`
- **Dynamic Fail Rules:** 
  - `fail_after_days`: ฐานของเวลาถือครอง (เบส 30 วัน)
  - `extend_days_per_appointment`: จำนวนวันที่ยืดให้ต่อ 1 นัดหมาย
  - `extend_days_sales_amount_threshold`: ยอดขายสะสมขั้นต่ำที่ต้องทำได้เพื่อรับการต่อเวลา
  - `extend_days_sales_reward`: จำนวนวันที่บวกเพิ่มให้เมื่อยอดขายถึงเกณฑ์
  - `max_total_days`: เพดานวันสุงสุด
- **Re-Evaluate Rules:** 
  - `on_fail_reevaluate`: หากหมดอายุ ให้วิ่งเข้าการประเมินถังอัตโนมัติ แทนที่จะย้ายไปถังเป้าหมายตายตัว
  - `blocked_target_baskets`: ระบุตะกร้าที่ **ห้าม** ย้ายไป (ใช้บล็อกข้ามสาย เช่น ห้ามลูกค้าสาย A ไปโผล่สาย B)

## 💡 Best Practices การแก้ไขไฟล์นี้
- ไฟล์มีขนาดใหญ่มาก (> 2000 บรรทัด) เพราะรวมระบบ Audit ไว้ 4 ระบบในไฟล์เดียว
- หากจะแก้ UI ของตาราง Audit ให้มองหา `activeGroup === 'ชื่อกลุ่ม'`
- โค้ดของ **Modal เพิ่ม/แก้ไขตะกร้า** จะอยู่ด้านล่างสุดของไฟล์ (`{editingBasket && (...) }`)
