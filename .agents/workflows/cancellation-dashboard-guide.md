# 📊 คู่มือการใช้งาน: แดชบอร์ดยกเลิก (Cancellation Dashboard)

หน้าแดชบอร์ดยกเลิก เป็นเครื่องมือสำหรับติดตามและวิเคราะห์สถิติคำสั่งซื้อที่ถูกยกเลิก (Cancelled Orders) ของพนักงานขาย (Telesale) แต่ละคน รวมถึงแสดงสัดส่วนสาเหตุการยกเลิก เพื่อให้นำข้อมูลไปปรับปรุงกระบวนการขายได้ นอกจากนี้ยังมีระบบให้ Supervisor กด "รับทราบ" เมื่อเกิดการยกเลิกขึ้น

หน้าจอถูกออกแบบมาให้แบ่งการทำงานออกเป็น **2 Tabs หลัก** เพื่อความสะอาดของ UI และความรวดเร็วในการใช้งาน

---

## 🎯 วัตถุประสงค์หลัก
1. **ติดตามอัตราการยกเลิก (Cancellation Rate):** วิเคราะห์ว่าพนักงานแต่ละคนมีสัดส่วนการยกเลิกเทียบกับออเดอร์ทั้งหมดเท่าไร
2. **สรุปมูลค่าความเสียหาย:** คำนวณมูลค่ารวมของออเดอร์ที่ถูกยกเลิกไปในแต่ละช่วงเวลา
3. **วิเคราะห์สาเหตุ:** แสดงกราฟสัดส่วน (Pie Chart) ของเหตุผลที่ลูกค้ายกเลิก
4. **วิเคราะห์ตามช่วงเวลา (Timeline):** แสดงข้อมูลสถิติแยกตามรายวัน รายเดือน และรายปี อย่างละเอียด
5. **ระบบรับทราบ (Acknowledgement):** แจ้งเตือนให้ Supervisor ทราบว่ามีการยกเลิกเกิดขึ้น และต้องเข้ามากดปุ่ม "รับทราบ" เพื่อติดตามปัญหา

---

## 👥 สิทธิ์การเข้าถึงข้อมูล (Role-based Visibility)
การแสดงผลข้อมูลจะแตกต่างกันไปตาม Role ของผู้ใช้งาน:
*   **Telesale (Role 7):** จะเห็นข้อมูลและสถิติ **เฉพาะของตัวเองเท่านั้น** ไม่สามารถเห็นของพนักงานคนอื่นได้ รวมไปถึงไม่เห็น Tab รับทราบออเดอร์
*   **Supervisor (Role 6):** จะเห็นข้อมูลของ **ตัวเอง และลูกทีมทุกคน** ในทีมที่ดูแล และสามารถเห็น Tab รอรับทราบ (Pending)
*   **Admin / Super Admin / Management:** จะเห็นข้อมูลของ **พนักงานทุกคน** ในระบบแบบภาพรวม

---

## 🖥️ ส่วนประกอบของหน้าจอ

### 1. เครื่องมือด้านบน (Top Bar & Tab Navigation)
*   **Date Range Picker (ตัวกรองเวลา):** เลือกช่วงวันที่ที่ต้องการดูข้อมูลออเดอร์ยกเลิก
*   **ปุ่ม Export Excel:** ส่งออกข้อมูลสถิติรายบุคคลออกมาเป็นไฟล์ `.xlsx` (ปรากฏเฉพาะเมื่ออยู่ใน Tab ภาพรวมสถิติ)
*   **Tab เมนูนำทาง:** เลือกเพื่อสลับการแสดงผลระหว่าง **"ภาพรวมสถิติ"** และ **"รอรับทราบ (Pending)"**

---

### Tab 1: 📈 ภาพรวมสถิติ (Stats Tab)
หน้าสำหรับผู้บริหารและหัวหน้าทีมในการดูข้อมูลเพื่อวิเคราะห์แนวโน้ม

1. **Summary Cards (การ์ดสรุปภาพรวม)**
   *   ออเดอร์ยกเลิกทั้งหมด
   *   อัตรายกเลิกเฉลี่ย (%)
   *   มูลค่าความเสียหายรวม

2. **สถิติการยกเลิกตามช่วงเวลา (Timeline Data Table)**
   *   ตารางรูปแบบ Excel ที่สามารถกดสลับโหมดมุมมองได้ 3 แบบ: **รายวัน (Daily)**, **รายเดือน (Monthly)**, และ **รายปี (Yearly)**
   *   แสดงวันที่/เดือน/ปี, จำนวนออเดอร์ที่ถูกยกเลิก, และมูลค่าความเสียหายรวมในคาบเวลานั้น
   *   สามารถ Scroll ขึ้นลงในตารางได้ หากมีจำนวนวันหลายวัน

3. **สถิติรายพนักงาน (Employee Stats)**
   *   ตารางแจกแจงข้อมูลสถิติพนักงานแต่ละคน (ออเดอร์รวม, ยอดยกเลิก, อัตรายกเลิก, และความเสียหาย) 

4. **สัดส่วนเหตุผลการยกเลิก (Pie Chart)**
   *   กราฟวงกลมด้านขวาล่างที่จะแสดงสรุปเหตุผลที่เกิดการยกเลิก

---

### Tab 2: ✅ รอรับทราบ (Pending Tab)
หน้าสำหรับการทำงาน (Operation) โดยเฉพาะ เพื่อเคลียร์ออเดอร์ที่ยกเลิกและยังไม่ได้รับรู้

*   **ตารางรวบรวมงาน:** จะดึงข้อมูลเฉพาะออเดอร์สถานะ Cancelled ที่ `is_acknowledged = 0` ขึ้นมาแสดงเป็นตารางแบนราบ (Flat List) ทำให้หาออเดอร์ค้างได้ง่าย
*   **ความปลอดภัยของข้อมูล:** ข้อมูลที่ดึงมาจะผ่านตัวกรองตาม Role ของผู้ใช้งาน (ตัวอย่าง: Supervisor จะเห็นเฉพาะออเดอร์รอรับทราบของตัวเองและลูกทีมเท่านั้น)
*   **ระบบกดรับทราบ (Acknowledge Button):** 
    *   **ปุ่มรายออเดอร์:** สามารถเลื่อนดูแล้วกด "รับทราบ" เป็นรายตัวได้
    *   **✨ ฟีเจอร์ Acknowledge All (รับทราบทั้งหมด):** ปุ่มสีน้ำเงินด้านขวาบน ระบบจะรวบรวม Order ID ทั้งหมดที่ปรากฏบนหน้าจอนั้น ส่งไป Stamp ว่ารับทราบแล้วในคลิกเดียว (รวดเร็วมาก)
*   ออเดอร์ที่ถูกรับทราบแล้ว จะหายไปจากตารางทันที (เหมือนระบบเคลียร์ Task)

## 🛠️ ข้อมูลสำหรับนักพัฒนา (Technical Details)

*   **Frontend Page:** `pages/CancellationDashboardPage.tsx`
*   **Sub-Components (ในโฟลเดอร์ `components/cancellation/`):** 
    *   `CancellationStatsTab.tsx`: รับผิดชอบ UI หน้าแดชบอร์ดสถิติ
    *   `CancellationAcknowledgeTab.tsx`: รับผิดชอบ UI หน้ารอรับทราบ
*   **API Services:** `services/api.ts` (`getCancellationDashboardStats`, `getUnacknowledgedCancellations`, `acknowledgeCancellation`, `acknowledgeAllCancellations`)
*   **Backend APIs:**
    *   `api/Orders/cancellation_dashboard_stats.php`: คิวรีข้อมูล Aggregation (นับพนักงาน, เหตุผล, และสร้าง Timeline แบบ วัน/เดือน/ปี)
    *   `api/Orders/get_unacknowledged_cancellations.php`: ดึงรายการออเดอร์ยกเลิกที่ `is_acknowledged` เป็น NULL หรือ 0 พร้อมรองรับ Role visibility
    *   `api/Orders/acknowledge_cancellation.php`: อัปเดตสถานะรับทราบทีละออเดอร์
    *   `api/Orders/acknowledge_all_cancellations.php`: อัปเดตสถานะรับทราบแบบกลุ่ม (Batch) โดยรับ Array ของ Order IDs
*   **Database Tables:** 
    *   `orders`, `order_cancellations`, `cancellation_types`
