# 📊 คู่มือการใช้งาน: แดชบอร์ดยกเลิก (Cancellation Dashboard)

หน้าแดชบอร์ดยกเลิก เป็นเครื่องมือสำหรับติดตามและวิเคราะห์สถิติคำสั่งซื้อที่ถูกยกเลิก (Cancelled Orders) ของพนักงานขาย (Telesale) แต่ละคน รวมถึงแสดงสัดส่วนสาเหตุการยกเลิก เพื่อให้นำข้อมูลไปปรับปรุงกระบวนการขายได้ นอกจากนี้ยังมีระบบให้ Supervisor กด "รับทราบ" เมื่อเกิดการยกเลิกขึ้น

---

## 🎯 วัตถุประสงค์หลัก
1. **ติดตามอัตราการยกเลิก (Cancellation Rate):** วิเคราะห์ว่าพนักงานแต่ละคนมีสัดส่วนการยกเลิกเทียบกับออเดอร์ทั้งหมดเท่าไร
2. **สรุปมูลค่าความเสียหาย:** คำนวณมูลค่ารวมของออเดอร์ที่ถูกยกเลิกไปในแต่ละช่วงเวลา
3. **วิเคราะห์สาเหตุ:** แสดงกราฟสัดส่วน (Pie Chart) ของเหตุผลที่ลูกค้ายกเลิก
4. **ระบบรับทราบ (Acknowledgement):** แจ้งเตือนให้ Supervisor ทราบว่ามีการยกเลิกเกิดขึ้น และต้องเข้ามากดปุ่ม "รับทราบ"

---

## 👥 สิทธิ์การเข้าถึงข้อมูล (Role-based Visibility)
การแสดงผลข้อมูลจะแตกต่างกันไปตาม Role ของผู้ใช้งาน:
*   **Telesale (Role 7):** จะเห็นข้อมูลและสถิติ **เฉพาะของตัวเองเท่านั้น** ไม่สามารถเห็นของพนักงานคนอื่นได้
*   **Supervisor (Role 6):** จะเห็นข้อมูลของ **ตัวเอง และลูกทีมทุกคน** ในทีมที่ดูแล
*   **Admin / Super Admin / Management:** จะเห็นข้อมูลของ **พนักงานทุกคน** ในระบบ

---

## 🖥️ ส่วนประกอบของหน้าจอ

### 1. เครื่องมือด้านบน (Top Bar)
*   **Date Range Picker (ตัวกรองเวลา):** สามารถเลือกช่วงวันที่ที่ต้องการดูข้อมูลออเดอร์ยกเลิก (ค่าเริ่มต้นคือไม่ได้ระบุ จะดึงข้อมูลทั้งหมด หรือตามที่ตั้งค่าเริ่มต้นไว้)
*   **ปุ่ม Export Excel:** สามารถส่งออกข้อมูลสถิติรายบุคคล (รหัสพนักงาน, ชื่อ, ออเดอร์ทั้งหมด, จำนวนที่ยกเลิก, อัตรายกเลิก, มูลค่า) ออกมาเป็นไฟล์ `.xlsx` ได้ทันที

### 2. Summary Cards (การ์ดสรุปภาพรวม)
แสดงสรุปภาพรวมตามช่วงเวลาที่เลือก (และตามสิทธิ์การมองเห็น):
*   **ออเดอร์ยกเลิกทั้งหมด:** จำนวนรายการคำสั่งซื้อที่เปลี่ยนสถานะเป็น ยกเลิก (Cancelled) 
*   **อัตรายกเลิกเฉลี่ย (%):** ค่าเฉลี่ยของเปอร์เซ็นต์การยกเลิกจากพนักงานทั้งหมดที่แสดงในตาราง
*   **มูลค่าความเสียหายรวม:** ผลรวมของยอดเงิน (`totalAmount`) ของออเดอร์ที่ถูกยกเลิกทั้งหมด

### 3. สถิติรายพนักงาน (Data Table)
ตารางแจกแจงข้อมูลสถิติของพนักงานขายแต่ละคน ประกอบด้วย:
*   ชื่อพนักงาน
*   ออเดอร์รวม (Total Orders)
*   ยอดที่ยกเลิก (Cancelled Count)
*   อัตราการยกเลิก (%) — *ไฮไลท์สีแดงหากอัตรา > 20%, สีส้ม > 10%*
*   มูลค่าความเสียหายรวมของพนักงานคนนั้น

**⭐ ฟีเจอร์ Drill-down:** เมื่อคลิกที่แถวของพนักงาน จะขยายออกมาแสดง **"รายการออเดอร์ที่ถูกยกเลิก"** รายออเดอร์ (ระบุรหัสออเดอร์, ชื่อลูกค้า, วันที่สั่งซื้อ, ยอดเงิน)

### 4. กราฟสาเหตุการยกเลิก (Cancellation Reasons Pie Chart)
แสดงกราฟวงกลมสรุปสัดส่วนของเหตุผลการยกเลิก (อ้างอิงข้อมูลจากตาราง `cancellation_types`) เพื่อให้ผู้บริหารหรือหัวหน้าทีมเห็นภาพรวมได้ชัดเจนว่าปัญหาหลักคืออะไร

## ✅ ระบบกดรับทราบออเดอร์ยกเลิก (Acknowledgement System)

เมื่อมีออเดอร์ถูกยกเลิก ระบบจะบันทึกรายการลงในแดชบอร์ด โดยมีระบบแจ้งเตือนและติดตามการรับทราบดังนี้:

*   **สัญลักษณ์แจ้งเตือน (Badge):** หากพนักงานคนไหนมีออเดอร์ยกเลิกที่ยังไม่ได้กดรับทราบ จะมีป้ายสีแดง `รอรับทราบ (X)` แสดงที่หลังชื่อพนักงานคนนั้น
*   **ปุ่มกดรับทราบ (Acknowledge Button):** 
    *   เมื่อขยายดูรายการออเดอร์ย่อย หากออเดอร์นั้นยังไม่ถูกรับทราบ จะมีปุ่ม **"กดรับทราบ" (Acknowledge)** 
    *   **สิทธิ์การกด:** เฉพาะ **Supervisor, Admin และ Super Admin** เท่านั้นที่มีสิทธิ์กดรับทราบ
    *   **Telesale:** จะเห็นสถานะเป็น "รอรับทราบ" แต่ไม่สามารถกดเองได้
*   **การกดรับทราบที่รวดเร็ว (Smooth Workflow):** เมื่อกดปุ่มรับทราบ ระบบจะทำการบันทึกข้อมูลไปยัง API ทันที **โดยไม่มีหน้าต่าง Confirm มากวนใจ** เพื่อความรวดเร็วในการทำงาน พร้อมแสดง Toast แจ้งเตือนผลลัพธ์
*   เมื่อกดรับทราบแล้ว: ระบบจะเปลี่ยนสถานะเป็นสัญลักษณ์ ✔️ **"รับทราบแล้ว"** ทันที และตัวเลขแจ้งเตือนหลังชื่อพนักงานจะลดลง

## 🛠️ ข้อมูลสำหรับนักพัฒนา (Technical Details)

*   **Frontend Page:** `pages/CancellationDashboardPage.tsx`
*   **Components:** 
    *   `OrderDetailModal.tsx`: สำหรับเปิดดูรายละเอียดออเดอร์เมื่อคลิกที่รหัสออเดอร์
    *   `Toast.tsx`: แจ้งเตือนผลการรับทราบออเดอร์
*   **API Services:** `services/api.ts` (`getCancellationDashboardStats`, `acknowledgeCancellation`, `listOrders`, `getOrderCancellationsBatch`)
*   **Backend APIs:**
    *   `api/Orders/cancellation_dashboard_stats.php`: คิวรีข้อมูลแบบ Aggregation ด้วย SQL (นับจำนวนออเดอร์รวม, นับเฉพาะที่สถานะเป็น 'Cancelled', รวมมูลค่า, Join ประเภทการยกเลิก, และคำนวณแจ้งเตือนที่ยังไม่ได้ acknowledge) พร้อมใช้ Logic ตรวจสอบ Role ของผู้เรียกใช้งาน
    *   `api/Orders/acknowledge_cancellation.php`: ทำหน้าที่รับ Request ไป อัปเดตสถานะ (`is_acknowledged = 1`, `acknowledged_by = {user_id}`, `acknowledged_at = NOW()`) ในตาราง `order_cancellations` (รองรับ payload ทั้ง `orderId` และ `order_id`)
    *   `api/Orders/get_order_cancellations_batch.php`: ดึงข้อมูลสถานะการรับทราบ (is_acknowledged) รายออเดอร์ มาประกอบร่างกับรายการออเดอร์ดิบ
*   **Database Tables:** 
    *   `orders` (คำนวณสถิติและดึงยอดเงิน)
    *   `order_cancellations` (จัดเก็บข้อมูลประเภทยกเลิก และสถานะการ Acknowledge)
    *   `cancellation_types` (ชื่อประเภทการยกเลิก)
