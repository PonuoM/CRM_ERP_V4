# Commit Helper

เมื่อได้รับมอบหมายให้ช่วยร่าง Commit Message ให้ดำเนินการดังนี้:

1. ตรวจสอบการเปลี่ยนแปลงของไฟล์ในโปรเจกต์
2. สร้าง Commit Message ตามหลักการ **Conventional Commits**:
   - `feat:` สำหรับฟีเจอร์ใหม่
   - `fix:` สำหรับการแก้บั๊ก
   - `docs:` สำหรับการปรับปรุงเอกสาร
   - `refactor:` สำหรับการจัดระเบียบโค้ดใหม่
   - `chore:` สำหรับงานทั่วไป (เช่น อัปเดต dependency)
3. ร่างให้ 2-3 แบบ (แบบสั้น, แบบละเอียด) และเตรียมให้พร้อมสำหรับการ copy & paste
