# 🎧 คู่มือระบบตรวจสอบสายโทรของคำสั่งซื้อ (Order Call Audit Guide)

คู่มือนี้กำหนดบทบาท ขั้นตอน และแนวทางการตรวจสอบความถูกต้องของคำสั่งซื้อ (Order) ร่วมกับไฟล์เสียงการสนทนา (Voice Recording) และข้อมูลในฐานข้อมูล เพื่อวิเคราะห์ความผิดปกติและการสวมสิทธิ์การขาย

## 📝 ขั้นตอนการทำงาน (Step-by-Step Instructions)

### 🔍 Step 1: ค้นหาไฟล์เสียงและข้อมูลเบื้องต้น

> [!TIP]
> เพื่อความรวดเร็วและแม่นยำสูงสุด แนะนำให้เรียกใช้งานไฟล์สคริปต์อัตโนมัติจากโฟลเดอร์ [query_audit/](file:///c:/laragon/www/CRM_ERP_V4/query_audit/) (เช่น รันผ่านบราวเซอร์ `http://localhost/CRM_ERP_V4/query_audit/query_order_audit.php?order_id=รหัสออเดอร์&mode=full`) เพื่อสืบค้นข้อมูลทั้งหมดในครั้งเดียว

เมื่อได้รับรหัสคำสั่งซื้อ (`orders.id`) ให้ดำเนินการดังนี้:
1. Query ข้อมูลของคำสั่งซื้อจากตาราง `orders` เพื่อหาข้อมูลเบื้องต้น ได้แก่:
   * วันที่สร้างออเดอร์ (`order_date`)
   * รหัสผู้สร้างออเดอร์ (`creator_id`)
   * รหัสลูกค้า (`customer_id`)
2. ดึงข้อมูลเบอร์โทรศัพท์ของลูกค้า และเบอร์โทรศัพท์รวมถึงชื่อจริงของ Agent ผ่านตาราง `customers` และ `users`
3. ค้นหาไฟล์เสียงที่บันทึกการคุยระหว่าง Agent กับลูกค้าในช่วงวันที่มีการสั่งซื้อ (อิง Logic การโทรเข้า/ออกและเบอร์โทรศัพท์จาก `CallHistoryPage.tsx`)

> [!IMPORTANT]
> **เงื่อนไขสำคัญด้านอายุไฟล์เสียง**: ไฟล์เสียงในระบบ OneCall จะถูกบันทึกและเก็บรักษาไว้เพียง **30 วันย้อนหลัง** เท่านั้น (ช่วง -30 วัน ถึง ปัจจุบัน)
> * **หากไม่พบไฟล์เสียง** (เนื่องจากอายุออเดอร์เก่ากว่า 30 วัน หรือระบบไม่บันทึก): ให้**หยุดการทำงานทันที** และแจ้งผู้ใช้ด้วยประโยคนี้:
>   > *"ไม่พบไฟล์เสียงในระบบที่ตรงกับเงื่อนไข กรุณาค้นหาและส่งไฟล์เสียงมาให้ฉันโดยตรงเพื่อดำเนินการต่อ"*

### 🔎 Step 2: สรุปไฟล์เสียงและตรวจสอบข้อมูล Database (Cross-check)

เมื่อพบไฟล์เสียงในระบบ หรือได้รับไฟล์เสียงจากผู้ใช้โดยตรง ให้ตรวจสอบและทำการวิเคราะห์ดังนี้:

#### 1. ตรวจสอบการนัดหมาย (Appointment Check)
* ฟังบริบทในบทสนทนาว่ามีการนัดหมายในอนาคตหรือไม่ เช่น *"เดี๋ยวพรุ่งนี้โทรหาใหม่ช่วงบ่ายนะ"* หรือ *"ไว้สัปดาห์หน้าติดต่อมาใหม่"*
* ไป Query ตาราง `appointments` เพื่อดูว่า Agent มีการบันทึกการนัดหมายตามข้อตกลงในสายสนทนาไว้จริงหรือไม่
* **จุดสังเกต**: หากในสายคุยมีการนัดหมาย แต่ไม่มีการสร้าง record ใน `appointments` หรือสร้างข้อมูลไม่ตรงความจริง ให้บันทึกเป็นข้อบกพร่อง

#### 2. ตรวจสอบความตรงกันของเบอร์โทรผู้ขาย (Phone Mismatch)
* ตรวจสอบว่าเบอร์โทรของ Agent ที่ผูกกับระบบบันทึกเสียงสนทนา (จาก OneCall API หรือ `localParty` ของสายโทร) ตรงกับเบอร์โทรศัพท์ในตาราง `users` ของผู้สร้างคำสั่งซื้อ (`orders.creator_id`) หรือไม่
* **หากไม่ตรงกัน**: ให้รายงานข้อสังเกตดังนี้:
  > *"เบอร์โทรไม่ตรงกัน: คาดว่าเป็นคนละคนกัน เช่น นาย A เป็นคนคุยกับลูกค้า แต่นาย B เป็นคนสร้างคำสั่งซื้อ"*

#### 3. ตรวจสอบการสวมสิทธิ์ผู้ขาย (Name Mismatch)
* ตั้งใจฟังชื่อเสียงแนะนำตัว of Agent ตอนเปิดสายสนทนา เช่น *"สวัสดีค่ะ [ชื่อพนักงาน] จาก..."*
* นำชื่อที่แนะนำตัวมาเปรียบเทียบกับชื่อจริง (`first_name`) ของผู้สร้างออเดอร์ในตาราง `users` (`orders.creator_id`)
* **หากชื่อไม่ตรงกัน**: ให้รายงานข้อสังเกตดังนี้:
  > *"ชื่อผู้พูดไม่ตรงกับชื่อผู้สร้างออเดอร์: อาจมีการใช้เบอร์หรือบัญชีคนอื่นในการขาย (เช่น หัวหน้าช่วยขายของให้ลูกน้อง ซึ่งเป็นพฤติกรรมที่ไม่ควรทำ)"*

## 💡 ตัวอย่าง SQL Queries ที่แนะนำสำหรับใช้ตรวจสอบ

### 1. ค้นหาข้อมูลออเดอร์ ผู้สร้าง และลูกค้า
```sql
SELECT 
    o.id AS order_id, 
    o.order_date, 
    o.creator_id, 
    u.first_name AS creator_name, 
    u.phone AS creator_phone,
    o.customer_id, 
    c.phone AS customer_phone
FROM orders o
LEFT JOIN users u ON o.creator_id = u.id
LEFT JOIN customers c ON o.customer_id = c.id
WHERE o.id = 'YOUR_ORDER_ID';
```

### 2. ตรวจสอบนัดหมายการโทรของลูกค้า
```sql
SELECT 
    id, 
    customer_id, 
    title, 
    notes, 
    due_at, 
    status, 
    created_by 
FROM appointments 
WHERE customer_id = 'YOUR_CUSTOMER_ID' 
  AND due_at >= 'YOUR_ORDER_DATE'
ORDER BY due_at DESC;
```
